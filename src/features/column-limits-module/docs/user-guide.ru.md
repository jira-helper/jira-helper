# Групповые лимиты WIP по колонкам (CONWIP)

| | |
|---|---|
| Где настраивается | Настройки доски → Колонки → Column group WIP limits |
| Где видно | Доска (детальный вид) |
| Хранилище | Свойство доски Jira (`subgroupsJH`) |

## Цель

Объединить несколько колонок под одним общим лимитом незавершённой работы, чтобы карточки во всех этих колонках учитывались в едином счётчике. Возможна фильтрация по swimlane и типам задач, плюс наглядное выделение групп цветом и предупреждения о превышении.

## Как настроить

1. Откройте **Настройки доски** для Scrum- или Kanban-доски (требуются права на изменение конфигурации доски).
2. Перейдите на вкладку **Колонки**.
3. Нажмите **Column group WIP limits**.
4. В диалоговом окне перетащите колонки в именованные группы. Для каждой группы задайте **лимит**, при необходимости — **цвет**, **типы задач** и область по **swimlane**.
5. Нажмите **Save**, чтобы сохранить настройки для всех на этой доске, или **Cancel**, чтобы отменить.

Сохранять могут только администраторы доски или пользователи с правами на изменение конфигурации доски.

## Как использовать

- На доске каждая группа колонок показывает бейдж **текущее / лимит** в заголовке первой колонки группы.
- Заголовки колонок группы стилизуются цветом группы, чтобы группу было легко узнать.
- При превышении лимита затронутые области колонок подсвечиваются **красным**.
- Подсчёт задач соответствует настройкам статистики доски Jira (включая учёт подзадач).

## Сценарии использования

- «Хочу ограничить общий объём незавершённой работы в цепочке Разработка + Code Review + Тестирование одним общим лимитом.»
- «Хочу, чтобы баги в колонках разработки считались отдельно от фиче-задач.»
- «Хочу исключить expedite-дорожку из определённых групповых лимитов.»

## Устранение неполадок

- **Бейдж группы показывает ноль**: проверьте, что в колонках группы есть задачи, и что фильтры по swimlane и типам задач не исключают всё подряд.
- **Цвет не отображается**: проверьте, задан ли цвет для группы; если не задан, цвет подбирается автоматически по имени группы.
- **Настройки не сохраняются**: убедитесь, что у вас есть права на изменение конфигурации доски в Jira.
- **Подзадачи считаются некорректно**: расширение следует настройкам подсчёта задач на уровне доски Jira; измените их в общих настройках доски.
