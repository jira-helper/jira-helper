# Прогресс подзадач

| | |
|---|---|
| Где настраивается | Board Settings → вкладка Sub-tasks progress |
| Где видно | Доска (детальный вид карточек в отслеживаемых колонках) |
| Хранилище | Jira Board Property |

## Цель

Показывать на карточках доски составной прогресс (полосы и/или компактные счётчики) по дочерней и связанной работе — подзадачам, задачам под эпиком, связанным задачам и при необходимости внешним ссылкам — в разрезе To Do, In Progress, Done и Blocked.

## Как настроить

1. Откройте доску Jira и нажмите **Jira Helper** рядом с боковой панелью.
2. Откройте вкладку **Sub-tasks progress**.
3. Включите фичу. **Reset all settings** сбрасывает настройки вкладки к значениям по умолчанию.
4. В блоке **Tracked columns** выберите колонки, в которых показывать прогресс на карточках.
5. В секции **Counting** включите нужные источники: дети эпика, подзадачи, связанные задачи (с возможной фильтрацией по типу связи) и внешние ссылки.
6. Настройте **Grouping**: выберите поле группировки по умолчанию (проект, исполнитель, тип задачи и др.), задайте игнорируемые группы и создайте свои группы с именами, правилами, стилем (полоса или счётчик), цветами и видимостью.
7. Сопоставьте **статусы** Jira с четырьмя категориями прогресса (to do, in progress, done, blocked). Не сопоставленные статусы исключаются из подсчёта.
8. При необходимости включите обработку **Blocked/Flagged**, чтобы учитывать помеченные флагом задачи как заблокированные и показывать предупреждения на карточках.
9. Настройте **цвета** — выберите готовую схему или свою палитру.

Настройки задаются на доску и сохраняются пользователями с правом редактирования доски.

## Как использовать

- В отслеживаемых колонках каждая карточка показывает настроенные полосы прогресса и/или счётчики.
- Полоса прогресса отображает составные сегменты (зелёный = done, синий = in progress, серый = to do, красный = blocked) или одну сводную полосу.
- Числовые счётчики показывают количество задач в каждой категории статусов.
- Группы можно настроить на скрытие, когда всё выполнено, или на показ только незавершённых задач.
- Если включена обработка blocked/flagged, на карточке появляется текст предупреждения при наличии заблокированных или помеченных флагом связанных задач.

## Сценарии использования

- **Мониторинг тимлидом:** Видеть прогресс эпиков и stories прямо с доски, не открывая каждую задачу.
- **Распределение нагрузки:** Группировать прогресс по исполнителю или проекту для оценки загрузки.
- **Обнаружение узких мест:** Замечать заблокированные подзадачи по индикатору на родительской карточке.
- **Кросс-проектный трекинг:** Использовать связанные задачи для отслеживания прогресса между проектами.

## Устранение неполадок

- **Полоса прогресса не отображается:** Проверьте, что фича включена и колонка добавлена в список отслеживаемых.
- **Неверная категория статуса:** Проверьте сопоставление статусов с категориями прогресса в настройках.
- **Задачи не учитываются:** Убедитесь, что нужный источник (подзадачи, дети эпика, связи) включён и статусы сопоставлены.
- **Проблемы производительности:** Ограничьте количество отслеживаемых колонок и источников для снижения нагрузки на Jira API.
