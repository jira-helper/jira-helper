# Диаграмма Ганта

| | |
|---|---|
| Где настраивается | Страница задачи → кнопка Helper или шестерёнка в панели Gantt → вкладка Gantt Chart |
| Где видно | Страница задачи (классический вид, под вложениями) |
| Настройки действуют | Только для вас |(`jh-gantt-settings`) |

## Цель

Добавить на страницу задачи в классическом виде горизонтальную диаграмму Ганта, которая показывает все связанные задачи — подзадачи, дети эпика и связанные задачи — на общей временной шкале, чтобы видеть, когда выполняется каждая часть работы и как менялся её статус.

## Как настроить

1. Откройте любую задачу в классическом виде и нажмите **Helper** в тулбаре задачи, затем выберите вкладку **Gantt Chart**. Настройки также открываются по кнопке-шестерёнке в тулбаре диаграммы.
2. Выберите **Scope** для редактирования: *Global*, *This project* или *This project + issue type*. Используйте **Copy from…** для заполнения нового scope из существующего. При отображении применяется наиболее специфичный scope с настройками.
3. В секциях **Start of bar** и **End of bar** постройте приоритетный список источников (поля дат или переходы статусов). Используйте **Add fallback** и стрелки вверх/вниз для порядка; первый источник, давший дату для задачи, побеждает.
4. В секции **Issues** выберите категории для включения: подзадачи, дети эпика, связанные задачи. Для связей можно ограничить по типу и направлению.
5. Добавьте **Exclusion filters**, чтобы исключить задачи, которые никогда не должны попадать на диаграмму (задача исключается при совпадении любого фильтра).
6. Добавьте **Bar colour rules** для выделения задач по значению поля или JQL. Правила проверяются сверху вниз; первое совпадение побеждает.
7. Добавьте **Quick filters** — переиспользуемые чипсы-переключатели. Режим: *field value* или *JQL*. Встроенные чипсы *Unresolved* и *Hide completed* доступны всегда.
8. Выберите дополнительные поля для **всплывающей подсказки** и нажмите **Save**.

Настройки каскадируются: project+issue type → project → global. Настройки сохраняются между перезагрузками браузера.

## Как использовать

- Секция Gantt **по умолчанию свёрнута**. Нажмите, чтобы развернуть и загрузить диаграмму.
- **Масштаб и панорама:** Zoom колёсиком мыши, кнопки +/−, выбор интервала (*hours*, *days*, *weeks*, *months*), перетаскивание по диаграмме.
- **Всплывающая подсказка:** Наведите курсор на любую полосу, чтобы увидеть ключ задачи, описание, статус, даты и настроенные поля.
- **Разбивка по статусам:** Включите в настройках, чтобы разделить каждую полосу на цветные сегменты (серый = to do, синий = in progress, зелёный = done, красный = blocked).
- **Быстрые фильтры:** Используйте чипсы над диаграммой для фильтрации. Введите текст в строку поиска (текстовый или JQL-режим). Несколько чипсов комбинируются через AND.
- **Открыть в модальном окне:** Нажмите кнопку расширения для просмотра на весь экран с сохранением zoom и состояния тулбара.
- **Жёлтые предупреждения:** Появляются, когда диаграмма неполная — "No history for X of Y tasks" или "X tasks not on chart". Наведите для деталей.
- При вводе в поля Gantt горячие клавиши Jira подавляются.

## Сценарии использования

- **Планирование спринта:** Видеть все связанные задачи на временной шкале для оценки готовности и зависимостей.
- **Обзор статусов:** Использовать разбивку по статусам, чтобы видеть, сколько времени задачи провели в каждом состоянии.
- **Отслеживание зависимостей:** Включать связанные задачи для визуализации кросс-задачных зависимостей.
- **Командная синхронизация:** Использовать модальный вид на стендапах для прохода по временной шкале.
