# Флаг на панели задачи

## Обзор

Функция без настройки, которая подсвечивает отмеченные флагом задачи в иерархии задач. Любой связанный элемент — связанная задача, подзадача, задача эпика — имеющий флаг Jira, получает жёлтый фон и значок флага, что позволяет легко заметить заблокированную или требующую внимания работу.

**Флаг Jira** — это встроенное свойство задачи (поле **Flagged**), которое отмечает задачу как имеющую препятствие или требующую внимания. Он отображается как значок красного флага в интерфейсе Jira и обычно используется для блокировок, остановленных работ или высокоприоритетных оповещений.

**Зачем:** При просмотре родительской задачи со множеством дочерних, поиск отмеченных флагом требует сканирования каждой строки в поисках маленького значка флага. Эта функция делает отмеченные флагом элементы визуально заметными с помощью жёлтого фона и заметного значка, избавляя от необходимости сканировать каждую строку по отдельности.

**Где:** Автоматически на всех маршрутах — настройка не требуется. Работает в классическом представлении задачи, новом представлении задачи, боковых панелях доски и результатах поиска.

**Как это работает:**
- На каждой странице задачи расширение сканирует все видимые связанные элементы (подзадачи, задачи эпика, связанные задачи) на наличие свойства `Flagged = "Yes"`
- Отмеченные флагом элементы получают жёлтый фон (`#ffe9a8` в классическом представлении, `#fffae6` в новом представлении задачи) и значок 🚩
- Текущая открытая задача, если она отмечена флагом, также получает значок флага в области заголовка
- Работает автоматически — без настроек, без конфигурации, без прав на доску

## Поддерживаемые представления

| Представление | Подсветка флага | Значок флага | Примечания |
|---|---|---|---|
| Классическая страница задачи | Жёлтый фон строки | 🚩 рядом с ключом | Сканируется полная иерархия (подзадачи, задачи эпика, связанные задачи) |
| Новое представление задачи | Светло-жёлтый (`#fffae6`) фон строки | 🚩 рядом с ключом | Подзадачи и связанные задачи |
| Боковая панель доски | Жёлтый фон | 🚩 рядом с заголовком | Панель задачи, открытая с доски |
| Результаты поиска | Жёлтый фон | 🚩 рядом с ключом | Каждая отмеченная флагом строка в навигаторе задач |
| Бэклог | Не поддерживается | — | Карточки бэклога имеют ограниченный доступ к DOM |

## Использование

### Автоматически: настройка не требуется

**Цель:** Видеть отмеченные флагом элементы подсвеченными автоматически.

1. Откройте любую задачу (классическое или новое представление, боковая панель доски, результаты поиска).
2. Отмеченные флагом связанные элементы автоматически подсвечиваются жёлтым фоном.
3. Значок флага (🚩) появляется рядом с ключом или заголовком задачи.
4. Функция работает сразу — без настройки доски, без переключателей, без прав.

### Иерархия задач: флаги подсвечены на всех уровнях

**Цель:** Визуально определить отмеченную флагом работу на каждом уровне дерева задач.

1. На классической странице задачи сканируются развёрнутые подзадачи, задачи эпика и связанные задачи.
2. Элементы с активным флагом Jira получают:
   - **Жёлтый фон** (`#ffe9a8` классическое, `#fffae6` новое представление задачи)
   - **Значок флага** рядом с ключом/ссылкой задачи
3. Если **сама открытая задача** отмечена флагом, значок флага также появляется в области заголовка (рядом с приоритетом/типом) в классическом представлении.

## Часто задаваемые вопросы

### Если я сниму флаг с задачи — обновится ли подсветка?

Да, но не в реальном времени. Подсветка отражает состояние задачи на момент загрузки страницы. Если вы изменили флаг в другой вкладке и вернулись, **перезагрузите страницу** (F5), чтобы увидеть обновлённое состояние. Боковая панель доски перезагружается при повторном открытии панели задачи.

### Работает ли это на всех страницах Jira?

Работает в классическом представлении задачи, новом представлении задачи, боковых панелях доски и результатах поиска. **Не** работает в бэклоге, гаджетах дашборда или на портале Jira Service Management.

### Настраивается ли эта функция?

Нет — она намеренно не имеет настроек. Нет параметров для изменения, колонок для выбора, прав для управления. Если жёлтый фон отвлекает, функция [Размытие данных](/docs/features/data-blurring) размывает весь текст и изображения на странице — не только флаги, но и заголовки задач, ключи, аватары и описания. Используйте её во время презентаций, чтобы полностью скрыть конфиденциальные данные, а не для выборочного подавления подсветки флагов.

### Замедляет ли это загрузку страницы?

Нет. Функция использует наблюдение за DOM и лёгкие проверки свойств. Она не добавляет измеримой задержки для типичных страниц Jira (до нескольких сотен видимых связанных элементов). Для страниц с тысячами связанных элементов собственный рендеринг Jira доминирует во времени загрузки страницы.

## Устранение неполадок

### Отмеченные флагом элементы не подсвечены на классической странице задачи

- Страница задачи должна быть **полностью загружена** перед активацией подсветки. Если страница всё ещё загружает подзадачи (AJAX), дождитесь отрисовки всех строк.
- Если задача использует **пользовательскую тему или плагин**, переопределяющий фон строк таблицы, жёлтая подсветка может быть переопределена. Проверьте в DevTools браузера, применён ли стиль, но переопределён.
- Очень старые версии Jira Server (до 7.0) могут хранить флаги иначе — убедитесь, что ваш экземпляр Jira обновлён.

### Значок флага есть, но нет жёлтого фона

- Жёлтый фон применяется через CSS-класс. Если ваш экземпляр Jira имеет глобальную таблицу стилей, устанавливающую `!important` на фон строк, подсветка может быть подавлена. Это наиболее распространено на сильно кастомизированных экземплярах Jira Server.

### Функция не работает в новом представлении задачи

- Новое представление задачи (React-интерфейс Jira) использует более светлый жёлтый (`#fffae6`). Если его не видно, версия Jira могла изменить структуру DOM — проверьте, что расширение обновлено до последней версии.
