# Диаграмма Ганта

## Обзор

Интерактивная диаграмма Ганта на странице задачи в классическом представлении. Отображает связанные задачи — подзадачи, задачи эпика и связанные задачи — на общей временной шкале с разбивкой по статусам, настраиваемыми цветами полос, масштабированием/панорамированием и быстрыми фильтрами.

**Зачем:** Встроенное представление задачи в Jira показывает связанные задачи как плоский список без временно́го измерения. Диаграмма Ганта визуализирует всю связанную работу на общей временной шкале, позволяя с первого взгляда выявить пробелы в расписании, пересечения работ и зависимости.

**Где:** Откройте любую задачу в классическом представлении → разверните сворачиваемый раздел **Jira Helper Gantt** (под вложениями). Настройки доступны через значок шестерёнки на панели инструментов диаграммы.

**Как это работает:**
- Каждая связанная задача становится горизонтальной полосой, позиционируемой по датам начала/окончания
- Источники дат настраиваются с приоритетными резервными вариантами (поля дат, события истории статусов)
- Цвета полос задаются правилами: по значению поля или JQL-выражению
- Несколько уровней области действия: Глобальный, Этот проект, Этот проект + Тип задачи (наиболее специфичный побеждает)
- Быстрые фильтры, текстовый/JQL-поиск и масштабирование/панорамирование обеспечивают интерактивное исследование
- Опциональная разбивка по статусам раскрашивает сегменты полосы по времени, проведённому в каждом статусе рабочего процесса

## Необходимые условия

- Задача должна быть открыта в **классическом представлении** (сворачиваемый раздел Ганта недоступен в новом представлении задачи Jira)
- Требуется хотя бы одна связанная задача (подзадача, задача эпика или связанная задача) с настроенным источником дат для отображения полос
- Настройки хранятся в `localStorage` для каждого браузера — см. [Настройки (Локальное хранилище)](/docs/settings#local-storage-personal)

## Пользовательские сценарии

### Открытие диаграммы Ганта на странице задачи (классическое представление)

**Цель:** Просмотреть горизонтальную временную шкалу всех работ, связанных с текущей задачей.

1. Откройте любую задачу в классическом представлении.
2. Разверните сворачиваемый раздел **Jira Helper Gantt** (под вложениями).
3. Каждая связанная задача отображается в виде горизонтальной полосы с меткой ключа задачи и заголовком.

**Примечание по производительности:** Диаграммы с 200+ задачами могут отображаться 5–10 секунд и становиться медленными при масштабировании/панорамировании. Используйте фильтры исключения и быстрые фильтры, чтобы уменьшить количество видимых задач. Для расчёта дат требуется доступ к истории изменений — на экземплярах Jira Server/Data Center с большими журналами изменений первая загрузка может быть медленной при получении исторических изменений статусов.

### Навигация: колесо мыши, панорамирование, всплывающие подсказки, клик по полосам

**Цель:** Исследовать диаграмму — приближать/отдалять, прокручивать, просматривать детали задач.

1. **Масштабирование:** Колесо прокрутки или кнопки **+ / −** на панели инструментов.
2. **Панорамирование:** Клик и перетаскивание в любом месте диаграммы или использование полос прокрутки.
3. **Всплывающая подсказка:** Наведите курсор на любую полосу, чтобы увидеть полный ключ, заголовок и выбранные дополнительные поля (исполнитель, статус, приоритет, даты, пользовательские поля).
4. **Кликните** по полосе, чтобы выбрать её (визуальная подсветка).
5. Используйте **селектор интервала** на панели инструментов для переключения между *часами*, *днями*, *неделями*, *месяцами* (автоподбор при первой загрузке).
6. Нажмите **Открыть в модальном окне**, чтобы развернуть диаграмму на всё окно. Настройки масштаба, панорамирования и интервала сохраняются при входе и выходе из модального окна. Закройте модальное окно кнопкой ✕ или нажатием Escape.

### Настройка сопоставления дат (поля начала/окончания с приоритетными резервными вариантами)

**Цель:** Определить, как вычисляются даты начала и окончания полосы для каждой задачи.

1. Нажмите кнопку **шестерёнки** на панели инструментов диаграммы → вкладка **Диаграмма Ганта** → выберите область действия: *Глобально*, *Этот проект* или *Этот проект + тип задачи*.
2. В разделах **Начало полосы** и **Конец полосы** постройте приоритетный список источников (поля дат или события истории статусов, например, «вошла в *В работе*»).
3. Используйте **Добавить резервный вариант** для добавления записей и **стрелки вверх/вниз** для изменения порядка.
4. Для каждой задачи побеждает первый источник, давший дату; задачи без подходящего источника исключаются из диаграммы.

> **Идентификатор поля:** Идентификатор поля — это внутренний идентификатор Jira для пользовательского поля (например, `customfield_10010`). Его можно найти через REST API Jira (`/rest/api/2/field`) или проверив имя поля в разделе **Администрирование → Задачи → Пользовательские поля**.

### Настройка правил цвета полос (поле или JQL → цвет)

**Цель:** Выделить полосы по значению поля или JQL-выражению.

1. В той же панели настроек прокрутите до раздела **Правила цвета полос**.
2. Нажмите **Добавить правило**, выберите режим: **значение поля** (идентификатор поля + значение) или **JQL** (JQL-выражение).
3. Выберите цвет и нажмите **Сохранить**.
4. Правила оцениваются сверху вниз; побеждает первое совпадение. Меняйте порядок стрелками.

### Выбор, что включать: подзадачи, задачи эпика, связанные задачи, определённые типы связей

**Цель:** Управлять тем, какие связанные задачи отображаются на диаграмме.

1. В настройках → раздел **Задачи** отметьте категории для включения:
   - Подзадачи
   - Задачи эпика
   - Связанные задачи (опционально ограничить по типу и направлению связи).

### Фильтры исключения (поле или JQL)

**Цель:** Скрыть шумные задачи (например, отменённые, тривиальные) с диаграммы для области действия.

1. В настройках → раздел **Фильтры исключения** нажмите **Добавить фильтр**.
2. Выберите режим **значение поля** (идентификатор поля + значение) или **JQL**.
3. Задача, соответствующая любому из фильтров (логика ИЛИ), исключается из диаграммы.
4. Изменения применяются ко всем, кто использует эту область действия, после нажатия **Сохранить**.

### Быстрые фильтры: встроенные фильтры, пользовательские пресеты, текстовый/JQL-поиск

**Цель:** Интерактивно фильтровать полосы без изменения настроек.

1. Над диаграммой используйте **строку быстрых фильтров**:
   - **Поле поиска** — введите текст или переключитесь в **режим JQL** для JQL-выражений.
   - Встроенные чипы **Нерешённые** и **Скрыть завершённые** (всегда доступны).
   - **Пользовательские чипы** — сохраните JQL-выражение как многоразовый чип через **Сохранить как быстрый фильтр**.
2. Несколько чипов комбинируются через И.
3. Активные фильтры и текст поиска действуют только в рамках сессии (сбрасываются при перезагрузке); сохранённые пресеты чипов сохраняются.

### Каскад настроек: глобальный → проект → проект+тип задачи

**Цель:** Задать глобальные значения по умолчанию и переопределить их для проекта или проекта+типа задачи.

1. Откройте настройки → переключатель области действия вверху: *Глобально*, *Этот проект* или *Этот проект + тип задачи*.
2. Используйте **Копировать из…** для заполнения новой области действия из существующей.
3. При отображении побеждает наиболее специфичная область действия, для которой есть настройки.
4. При повторном открытии настроек переключатель области действия автоматически устанавливается на фактически применяемый уровень, так что вы никогда не редактируете пустую форму для более специфичной области.

### Разбивка по статусам (включение)

**Цель:** Разделить каждую полосу на цветные сегменты на основе фактической истории статусов.

1. В настройках включите **Разбивку по статусам**.
2. Полосы теперь показывают сегменты: серый (*к выполнению*), синий (*в работе*), зелёный (*готово*), красный (*заблокировано*).
3. Задачи с датой начала, но без даты окончания, отображаются от даты начала до **сегодняшнего дня** с маркером предупреждения (⚠️) на правом конце полосы, указывающим на активную незавершённую задачу.
4. Задачи без дат начала и окончания отображаются в сворачиваемом разделе **не на диаграмме** под диаграммой с указанием причины для каждой задачи.

## Устранение неполадок

### На диаграмме нет полос

- Убедитесь, что в настройках отмечена хотя бы одна категория в разделе **Задачи** (подзадачи, задачи эпика, связанные задачи)
- Проверьте, что для применяемой области действия настроены источники дат — посмотрите на переключатель области действия в настройках, чтобы увидеть, какой уровень активен
- Для связанных задач убедитесь, что фильтр типа связи (если задан) соответствует фактическим связям задачи
- Все настроенные источники дат могут быть пустыми для связанных задач — попробуйте добавить резервный вариант, например, пользовательское поле даты, которое последовательно заполняется

### Неправильные даты на полосах

- Проверьте порядок приоритетов в разделах **Начало полосы** / **Конец полосы** в настройках. Побеждает первый совпавший источник — если источник с более низким приоритетом точнее, переместите его выше в списке.
- Источники дат по истории статусов («вошла в *В работе*») требуют наличия переходов статусов в журнале изменений. Если задача была создана сразу в статусе «В работе», события «вошла» нет, и этот источник не даёт даты.
- Поля дат должны иметь тип **Date** или **DateTime** в Jira — текстовые поля, содержащие даты, не распознаются.

### Настройки не применяются / сбрасываются

- Настройки хранятся в `localStorage`. Очистка данных браузера для домена Jira удалит всю конфигурацию диаграммы Ганта.
- Если настройки, кажется, сбрасываются, проверьте, что вы редактируете правильную область действия (Глобально vs. Этот проект vs. Этот проект + Тип задачи). Более специфичная область может переопределять ту, которую вы редактировали.
- После изменения настроек нажмите **Сохранить** и перезагрузите страницу задачи для применения изменений.
