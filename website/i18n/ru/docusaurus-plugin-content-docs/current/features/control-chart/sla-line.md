# Линия SLA

## Обзор

Горизонтальная опорная линия на отчёте Jira **Control Chart** (Контрольная диаграмма), показывающая порог соглашения об уровне обслуживания (SLA). Отображает затенённую полосу, запись легенды и метку процентиля для задач, укладывающихся в целевой срок или быстрее.

**Зачем:** В Control Chart Jira нет встроенной линии SLA. Без неё приходится мысленно сравнивать каждую точку данных с целевым значением SLA — это медленно и подвержено ошибкам. Линия SLA добавляет постоянный визуальный порог, позволяя мгновенно видеть, сколько задач укладывается в SLA, а сколько превышает его.

**Где:** Отчёты → Control Chart (для любой доски) → поле SLA в области параметров диаграммы. Линия отображается как горизонтальное наложение на диаграмму.

**Как это работает:**
- Введите числовое значение SLA в днях
- Диаграмма обновляется при потере фокуса (когда вы кликаете вне поля SLA), добавляя горизонтальную опорную линию, затенённую полосу и метку процентиля
- Сохраните значение в свойствах доски, чтобы оно сохранялось для всех пользователей доски (требуются права на редактирование)
- Без прав на редактирование значение SLA действует только в рамках сессии

## Необходимые условия

- Доступ к разделу **Отчёты → Control Chart** доски (доступно на Scrum и Kanban досках)
- Для **сохранения** значения SLA: **права на редактирование доски** (хранится в [свойствах доски](/docs/settings#board-properties-team-shared))
- Без прав на редактирование: вы всё равно можете задать и использовать линию SLA, но она сбросится при уходе со страницы

## Пользовательские сценарии

### Установка значения SLA в днях

**Цель:** Увидеть, сколько задач укладывается в целевой временной интервал.

1. Перейдите на доску → **Отчёты** → **Control Chart**.
2. В области параметров диаграммы под диаграммой найдите поле **SLA**.
3. Введите числовое значение в **днях** (например, `10` для 10-дневного SLA).
   - Допустимые значения: положительные целые и десятичные числа (например, `1`, `7`, `14.5`). Отрицательные значения и ноль игнорируются.
   - Пустое поле убирает линию SLA.
4. Кликните или нажмите Tab **вне поля** (потеря фокуса). Диаграмма обновится:
   - Горизонтальная **опорная линия** на уровне значения SLA
   - **Затенённая полоса** под линией для быстрой визуальной оценки
   - Запись легенды с меткой **SLA**
   - **Метка процентиля**, показывающая долю завершённых задач, укладывающихся в порог или быстрее (например, `72%`)

### Сохранение SLA в свойствах доски (требуются права на редактирование)

**Цель:** Сохранить значение SLA, чтобы оно загружалось автоматически для всех, кто использует эту доску.

1. После установки значения SLA нажмите **Сохранить** под полем SLA.
2. Значение записывается в свойства доски Jira.
3. При последующих посещениях любым участником доски линия SLA восстанавливается автоматически.
4. Если у вас нет прав на **редактирование доски**, кнопка **Сохранить** скрыта — SLA действует только в рамках сессии и сбрасывается при уходе со страницы.

### Чтение диаграммы с линией SLA

**Цель:** Интерпретировать Control Chart с контекстом SLA.

1. Когда SLA задан, **Control Chart** показывает:
   - Горизонтальную **опорную линию** на уровне порога SLA (в днях)
   - **Затенённую полосу** под линией — точки данных в полосе укладываются в SLA
   - Запись легенды с меткой **SLA**
   - **Метку процентиля**, показывающую долю завершённых задач, соответствующих SLA (например, `85% из 142 задач`)
2. Точки данных выше линии превышают SLA — это задачи, которые заняли больше времени, чем целевой срок.
3. **Наведите курсор** на линию SLA, чтобы увидеть точное значение процентиля.

## Граничные случаи

| Сценарий | Поведение |
|---|---|
| SLA = 0 | Линия на нуле, процентиль = 0% (ни одна задача не выполняется за нулевое время). Затенённая полоса и легенда всё равно отображаются. |
| SLA = пусто | Линия SLA полностью убирается — диаграмма не показывает наложение SLA. |
| SLA = отрицательное (например, `-5`) | Значение игнорируется. Аналогично пустому. |
| 0% задач в пределах SLA | Линия отображается, метка процентиля показывает «0%». Затенённая полоса имеет нулевую высоту. |
| На доске нет завершённых задач | Метка процентиля показывает «N/A» — процентиль невозможно рассчитать без точек данных. |
| Десятичные значения (например, `2.5`) | Принимаются. Линия рисуется на 2,5 днях. Процентиль отражает точность до долей дня. |

## Устранение неполадок

### Кнопка «Сохранить» не видна

Кнопка **Сохранить** появляется только при наличии **прав на редактирование доски** в Jira. Без них вы всё равно можете использовать линию SLA в течение сессии, но значение не сохранится. Попросите администратора доски Jira установить значение SLA от вашего имени.

### Линия SLA не появляется после ввода значения

Линия обновляется **при потере фокуса** — кликните или нажмите Tab вне поля ввода SLA, чтобы запустить обновление. Линия не обновляется, пока вы активно печатаете.

### Процентиль показывает «N/A»

Это происходит, когда в Control Chart нет завершённых задач для расчёта. Измените временной диапазон диаграммы или убедитесь, что в выбранном периоде есть решённые задачи.

### Линия отображается в неправильной позиции

Значение SLA указывается в **календарных днях**, а не в рабочих. 5-дневный SLA рисует линию на отметке 5 дней на оси Y диаграммы. Убедитесь, что ваш целевой срок SLA выражен в календарных днях.
