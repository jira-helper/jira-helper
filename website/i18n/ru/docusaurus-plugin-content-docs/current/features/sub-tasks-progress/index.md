# Индикаторы прогресса на карточках

Функция «Прогресс подзадач» добавляет на карточки доски компактные индикаторы прогресса (и опционально числовые счётчики), показывающие статус выполнения дочерних задач: подзадач, задач эпика, связанных задач и внешних ссылок.

**Зачем:** Отслеживание прогресса эпика или задачи обычно требует открытия каждой задачи по отдельности. Встроенные индикаторы прогресса позволяют тимлидам и участникам команды видеть статус выполнения непосредственно на доске, с поддержкой группировки по исполнителю, проекту или другим полям.

**Где:** Настройки доски → Jira Helper → вкладка «Прогресс подзадач». Индикаторы прогресса отображаются на карточках в выбранных колонках.

**Как это работает:**
- Работа распределяется по четырём категориям прогресса: **todo** (к выполнению), **inProgress** (в работе), **done** (готово), **blocked** (заблокировано)
- Сопоставление статусов с категориями настраивается — вы решаете, какие статусы Jira считать «к выполнению», «в работе», «готово» или «заблокировано»
- Источники задач включаются независимо: подзадачи, задачи эпика, связанные задачи (с опциональным фильтром по типу связи), внешние ссылки
- Прогресс может отображаться как единая полоса или как многосегментная разделённая полоса
- **Группировка** (по исполнителю, проекту, автору, приоритету, типу задачи или пользовательским полям) разделяет полосу на цветные сегменты по группам
- Режимы отображения групп: цветные полосы, простые счётчики или и то, и другое; группы можно скрывать после завершения или показывать только с незавершённой работой
- Обнаружение заблокированных: флаги Jira и связи `is blocked by` (заблокировано) могут автоматически помечать задачи как заблокированные и показывать предупреждения на родительских карточках
- Настройки задаются для каждой доски и хранятся в свойствах доски

## Необходимые условия

- **Права на редактирование доски** требуются для настройки сопоставления статусов и источников (настройки хранятся в [свойствах доски](/docs/settings#board-properties-team-shared))
- Для отображения индикаторов прогресса доска должна иметь хотя бы одну колонку, выбранную в **настройках колонок**
- Для источников типа «связанные задачи эпика» задачи должны быть связаны **от** дочерних задач эпика **к** другим задачам через указанный тип и направление связи

## Пользовательские сценарии

### Включение функции

**Цель:** Включить индикаторы прогресса для текущей доски.

1. Откройте доску и нажмите **Jira Helper** на панели инструментов доски.
2. Выберите вкладку **Прогресс подзадач**.
3. Установите флажок **Включить** в верхней части панели.
4. Индикаторы прогресса теперь отображаются на карточках (при условии настройки колонок и источников).

### Настройка колонок для отображения индикаторов

**Цель:** Ограничить отображение индикаторов прогресса определёнными колонками рабочего процесса.

1. В настройках «Прогресс подзадач» найдите раздел **Настройки колонок**.
2. Выберите колонки, в которых должны отображаться индикаторы прогресса.
3. Карточки в невыбранных колонках не будут показывать индикаторы, даже если функция включена.

### Выбор источников задач

**Цель:** Определить, что считается «дочерней работой» для расчёта прогресса.

1. В настройках найдите раздел **Подсчёт**.
2. Включайте каждый источник независимо. Каждый включённый источник добавляет свои задачи в категории прогресса.

#### Задачи эпика

Учитывать задачи в том же эпике.

#### Связанные задачи эпика

Учитывать задачи, связанные с дочерними задачами эпика через определённые типы связей. Выберите **тип связи** (например, `blocks`, `relates to`) и **направление** (входящее = задачи, которые ссылаются *на* дочернюю задачу эпика; исходящее = задачи, на которые ссылается дочерняя задача эпика).

#### Подзадачи

Учитывать прямые подзадачи карточки в Jira.

#### Связанные задачи

Учитывать задачи, связанные с карточкой, с опциональной фильтрацией по типу и направлению связи.

#### Связанные задачи подзадач

Учитывать задачи, связанные с подзадачами, с опциональной фильтрацией по типу и направлению связи.

#### Внешние ссылки

Учитывать задачи, связанные с карточкой через внешние ссылки (URL-ссылки).

#### Внешние ссылки эпика

Учитывать внешние ссылки у дочерних задач эпика.

### Сопоставление статусов с категориями прогресса

**Цель:** Определить, какие статусы Jira считаются «к выполнению», «в работе», «готово» и «заблокировано».

1. В настройках найдите раздел **Сопоставление статусов с прогрессом**.
2. Для каждого статуса Jira в вашем рабочем процессе назначьте категорию:
   - **К выполнению:** ещё не начато
   - **В работе:** ведётся активная работа
   - **Готово:** завершено
   - **Заблокировано:** есть препятствия (опционально — блокировка также может определяться по флагам/связям)
3. Статусы, не сопоставленные ни с одной категорией, исключаются из расчётов.
4. Сопоставление учитывает стандартные категории статусов Jira, если не переопределено.

### Настройка группировки и пользовательских групп

**Цель:** Разделить индикатор прогресса на цветные сегменты по полю (например, по исполнителю или проекту).

1. В настройках найдите раздел **Группировка**.
2. Включите **Группировать по полю** для активации сегментации.
3. Выберите **Поле группировки** из выпадающего списка: `project`, `assignee`, `reporter`, `priority`, `creator`, `issueType` или пользовательское поле.
4. Добавьте **Игнорируемые группы** — конкретные значения поля, которые нужно исключить из индикатора (например, исключить определённого исполнителя, находящегося в отпуске).
5. Настройте параметры отображения:
   - **Показывать группы как счётчики:** отображать числовые значения вместо полос для каждой группы
   - **Скрывать завершённые:** скрывать группы, где вся работа выполнена
   - **Показывать только незавершённые:** отображать только группы с незавершённой работой
   - **Пользовательские группы:** задать именованные группы с явными фильтрами. Введите **название группы** (отображается на индикаторе) и **фильтр** в формате `fieldId = "value"` (например, `customfield_10010 = "Frontend"` или `priority = "High"`), затем выберите цвета
6. Для каждого значения поля группировки (или пользовательской группы) можно задать:
   - **Цвет незавершённой работы:** цвет полосы/сегмента для незавершённой работы
   - **Цвет завершённой работы:** цвет полосы/сегмента для завершённой работы

### Обнаружение заблокированных задач через флаги и связи

**Цель:** Автоматически отмечать заблокированные дочерние задачи и показывать предупреждение на родительской карточке.

1. В настройках найдите раздел **Заблокированные задачи**.
2. Включите **Определение блокировки по флагам**, чтобы помечать отмеченные флагом задачи как заблокированные.
3. Включите **Определение блокировки по связям** и выберите тип связи (например, `is blocked by`), чтобы помечать задачи с такой связью как заблокированные.
4. На доске родительские карточки с заблокированными дочерними задачами отображают индикатор ⚠️ с подсказкой (например, «2 заблокированные задачи»).

### Вид на доске: компактный цветной индикатор прогресса

**Цель:** Видеть статус выполнения непосредственно на карточках доски.

1. Посмотрите на карточки в выбранных колонках. Каждая карточка показывает индикатор прогресса под заголовком.
2. **Без группировки:** единая цветная полоса с сегментами: «к выполнению» (серый), «в работе» (синий), «готово» (зелёный) и «заблокировано» (красный). Числовые счётчики (например, `3/5`) также могут отображаться в зависимости от настроек.
3. **С группировкой:** полоса разделена на цветные сегменты, по одному на группу. Каждый сегмент показывает долю выполнения этой группы.
4. **Наведите курсор** на сегмент, чтобы увидеть подсказку с названием группы и счётчиками (например, `Алиса: 2/4 готово`).
5. Если включено обнаружение блокировок, на карточке появляется индикатор предупреждения (⚠️), когда любая дочерняя работа заблокирована или отмечена флагом, вместе с кратким текстом подсказки.
6. После обновления статусов задач индикатор прогресса обновляется при обновлении доски.

## Производительность

- Каждый активный источник задач генерирует API-запросы пропорционально количеству карточек × количество дочерних задач на карточку. На досках с **100+ карточек и 3+ активными источниками** индикаторы прогресса могут отображаться 3–5 секунд.
- Для больших эпиков (50+ дочерних задач) ограничьте активные источники только необходимыми данными. Отключите **Связанные задачи эпика** и **Внешние ссылки эпика**, если они не требуются.
- Группировка по пользовательским полям добавляет дополнительный API-запрос на карточку для чтения значений полей. Используйте стандартные поля (исполнитель, приоритет и т.д.) для лучшей производительности.

## Часто задаваемые вопросы

### Почему на моих карточках нет индикаторов прогресса?

Наиболее частые причины: (1) функция не включена для доски, (2) в **настройках колонок** не выбраны колонки, (3) в разделе **Подсчёт** не включены источники задач, (4) сопоставление статусов неполное — если все дочерние статусы не сопоставлены, индикатор показывает 0%.

### Что происходит, если у карточки нет подзадач или дочерних задач?

Индикатор прогресса не отображается на карточке. Индикатор показывается только при наличии хотя бы одной дочерней задачи, учтённой через включённые источники.

### Как исключить определённые статусы из индикатора прогресса?

В таблице **Сопоставление статусов с прогрессом** оставьте ячейку категории пустой для статусов, которые нужно игнорировать. Задачи в этих статусах исключаются из всех расчётов прогресса.

### Обновляется ли обнаружение блокировок автоматически при установке/снятии флага?

Да — статус блокировки, определённый по флагам Jira и связям задач, обновляется при обновлении доски. Поставьте флаг задаче в другой вкладке, вернитесь на доску, и индикатор ⚠️ появится на родительской карточке при следующем обновлении доски.

## Устранение неполадок

### Индикатор прогресса показывает 0% / полностью серый

- Проверьте, что хотя бы одна дочерняя задача имеет статус, сопоставленный с **Готово** или **В работе** в настройках сопоставления
- При использовании источников связанных задач убедитесь, что тип и направление связи заданы правильно — направление может быть инвертировано
- Связанные задачи эпика требуют, чтобы связь была у самой дочерней задачи эпика, а не у эпика

### Индикатор блокировки не отображается

- Убедитесь, что включён хотя бы один из параметров: **Определение блокировки по флагам** или **Определение блокировки по связям**
- Для обнаружения по связям проверьте, что выбранный тип связи существует в вашем экземпляре Jira
- Родительская карточка должна находиться в выбранной колонке

### Изменения сопоставления статусов не применяются

- Нажмите **Сохранить** после изменения сопоставления. Несохранённые изменения теряются при переключении вкладки или перезагрузке доски.
