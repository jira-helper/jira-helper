# Swimlane WIP Limits

## Обзор

Swimlane WIP Limits задают каждой строке swimlane собственный лимит незавершённой работы. Вы можете установить разные лимиты для разных дорожек — например, дорожку «Expedite» с низким лимитом и дорожку «Normal» с более высоким.

Это дополняет [Column Group WIP Limits](/docs/features/wip-limits/column-limits): вы можете использовать обе функции одновременно для многоуровневого контроля WIP. Когда обе активны, каждый лимит применяется независимо — задача может одновременно учитываться и лимитом swimlane, и лимитом группы колонок.

Функция отображается на виде **доски** (Board) и настраивается во вкладке **Настройки доски → Swimlanes**.

**Предварительные требования:**
- Для настройки лимитов требуются права **администратора доски**
- Стратегия swimlane в Jira должна быть установлена на **Custom** (функция не работает с Query-based, Assignee-based, Story-based или Epic-based swimlane)
- Каждая swimlane, которую вы хотите ограничить, должна существовать как пользовательская swimlane во вкладке Swimlanes Jira

## Пользовательские сценарии

### Установка WIP-лимита на swimlane

**Цель:** Назначить числовой WIP-лимит одной или нескольким swimlane.

1. Откройте **Настройки доски** → вкладка **Swimlanes**.
2. Убедитесь, что стратегия swimlane Jira установлена на **Custom** (эта функция требует пользовательских swimlane).
3. Нажмите **Configure WIP Limits** — эта кнопка появляется в области панели инструментов вкладки Swimlanes, рядом с селектором стратегии swimlane.
4. В модальном окне для каждой swimlane введите число **лимита**. Оставьте пустым для отсутствия лимита.
5. Нажмите **OK**, чтобы сохранить все изменения сразу.

### Выбор колонок, учитываемых в лимите

**Цель:** Ограничить подсчёт только определёнными колонками в пределах swimlane.

1. В модальном окне настройки WIP swimlane нажмите селектор **колонок** для swimlane.
2. По умолчанию выбрано **All columns** — учитываются все колонки.
3. Снимите выбор **All columns** и выберите конкретные колонки.
4. Только задачи в выбранных колонках учитываются в лимите этой swimlane.
5. Нажмите **OK** для сохранения.

### Фильтрация по типам задач

**Цель:** Учитывать только определённые типы задач в лимите swimlane.

1. В модальном окне настройки WIP swimlane откройте фильтр **типа задач** для swimlane.
2. Выключите **Count all issue types**.
3. Выберите конкретные типы задач (Bug, Story, Task и т.д.).
4. Если **не выбрано ни одного типа**, лимит для этой swimlane полностью отключается.
5. Нажмите **OK** для сохранения.

### Удаление или отключение лимита

**Цель:** Прекратить применение WIP-лимита для swimlane.

1. Откройте **Настройки доски** → вкладка **Swimlanes** и нажмите **Configure WIP Limits**.
2. Чтобы **удалить** лимит: очистите поле лимита для этой swimlane (удалите число так, чтобы поле стало пустым) и нажмите **OK**. Значок и контроль лимита исчезают.
3. Чтобы **временно отключить**: очистите поле лимита, сохраните и позже снова введите число, когда захотите включить обратно. Отдельного переключателя вкл/выкл нет — удаление значения лимита является единственным способом деактивации.
4. Нажмите **OK** для сохранения изменений.

### Вид доски: мониторинг загрузки swimlane

**Цель:** Видеть статус WIP каждой swimlane на доске.

1. Каждая swimlane с настроенным лимитом показывает значок **количество / лимит** на своём заголовке (например, `3 / 5`).
2. Когда swimlane **превышает** лимит, заголовок подсвечивается красным фоном.
3. Дорожки без настроенного лимита не показывают значок и ведут себя как обычно.
4. Подсчёт учитывает фильтры колонок и типов задач, которые вы сохранили.

## Граничные случаи

- **Ошибка не-Custom swimlane:** Если стратегия swimlane Jira установлена на что-либо отличное от Custom (например, Assignee-based), модальное окно отображает сообщение об ошибке с предложением переключиться на Custom swimlane. Лимиты нельзя настроить, пока вы этого не сделаете.
- **Отрицательные или нулевые лимиты:** Поле лимита принимает только положительные целые числа. Ввод 0 или отрицательного числа отклоняется. Используйте 1 как минимальный лимит.
- **Пустое поле лимита:** Если оставить поле лимита swimlane пустым, лимит для этой swimlane не применяется. Значок не отображается.
- **Swimlane удалена:** Если swimlane удалена с доски после того, как вы настроили для неё лимит, устаревшая конфигурация молча игнорируется. Настроенные лимиты сохраняются в свойствах доски, но не имеют эффекта.
- **Взаимодействие с Column Group WIP Limits:** Обе функции работают независимо. Задача может превышать один тип лимита, соблюдая другой. Красная подсветка может накладываться на доске.
- **Стратегия swimlane изменена после настройки:** Если лимиты swimlane настроены, а стратегия swimlane позже изменена с Custom на другой тип (например, Assignee-based), лимиты сохраняются в свойствах доски, но перестают применяться. Значки исчезают с доски. Возврат стратегии на Custom повторно активирует ранее настроенные лимиты без необходимости вводить их заново.

## Устранение неполадок

### Значки не отображаются на доске

1. Убедитесь, что стратегия swimlane установлена на **Custom** в Настройки доски → Swimlanes.
2. Убедитесь, что вы сохранили лимиты (нажали **OK** в модальном окне).
3. Убедитесь, что поле лимита не пустое для swimlane, на которой вы ожидаете значки.
4. Проверьте, что вы находитесь на виде детализации доски (`?view=detail`) — значки не отображаются в Backlog, Planning и других видах.
5. Обновите страницу доски после сохранения.

### Кнопка «Configure WIP Limits» отсутствует

1. Убедитесь, что у вас есть права **администратора доски**.
2. Убедитесь, что вы находитесь на вкладке **Swimlanes** настроек доски (не Columns или Card Layout).
3. Кнопка появляется только когда стратегия swimlane — **Custom**.

### Лимиты считаются некорректно

1. Проверьте **фильтры колонок** — если выбраны конкретные колонки, учитываются только они. Попробуйте «All columns» для проверки.
2. Проверьте **фильтры типов задач** — если выбраны конкретные типы, учитываются только они.
3. Задачи на под-swimlane или родительских swimlane учитываются по своей swimlane, а не агрегируются.
