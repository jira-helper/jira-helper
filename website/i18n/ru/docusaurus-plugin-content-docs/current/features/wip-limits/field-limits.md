# WIP-лимиты по значениям полей (Распределение ресурсов)

## Обзор

WIP-лимиты по значениям полей позволяют ограничить количество задач (или сумму числовых значений), сгруппированных по значению поля. Вместо подсчёта всех карточек в колонке вы определяете правила на основе полей, отображаемых на карточках — например, ограничить количество задач с «Приоритет = Критичный» до 2 или ограничить общее количество story points «В работе» до 20.

Используйте это для распределения ресурсов (лимиты скорости сжигания по командам, спринтам или уровням приоритета). Функция доступна на представлении **Доска** и настраивается в **Настройки доски → вкладка Макет карточки**.

**Необходимые условия:**
- Для настройки лимитов требуются права **администратора доски**
- Поле, которое вы хотите использовать, должно присутствовать в макете карточки (настраивается в Настройки доски → Макет карточки)
- Для режима **Сумма чисел** поле должно быть числовым (например, Story Points, Первоначальная оценка)

## Пользовательские сценарии

### Добавить правило: выбор поля и режима расчёта

**Цель:** Определить правило лимита на основе поля.

1. Откройте **Настройки доски** → вкладка **Макет карточки**.
2. Убедитесь, что нужное поле уже присутствует в макете карточки.
3. Нажмите **Редактировать WIP-лимиты по полям**.
4. Нажмите **Добавить лимит** и настройте:
   - **Поле** — выберите поле из макета карточки.
   - **Режим расчёта** — выберите способ учёта задач (см. таблицу ниже).
5. Для режимов **Точное значение** и **Любое из значений** введите соответствующие значения.
6. Установите число **Лимита**.
7. Нажмите **Добавить лимит**, затем **Сохранить** для применения.

### Краткий справочник режимов расчёта

| Режим | Что учитывается | Пример использования |
|---|---|---|
| **Поле имеет значение** | Задачи, в которых поле имеет любое значение (не пустое) | «Ограничить задачи с установленным Приоритетом до 5» |
| **Точное значение** | Задачи, точно соответствующие одному значению | «Ограничить баги с Приоритет = Критичный до 2» |
| **Любое из значений** | Задачи, соответствующие любому из нескольких значений | «Ограничить задачи для Команды Альфа или Команды Бета до 8» |
| **Сумма чисел** | Сумма числовых значений поля вместо количества задач | «Ограничить общие story points в работе до 20» |

### Управление, редактирование и удаление правил

**Цель:** Изменить или удалить существующее правило лимита по полю.

1. Откройте **Настройки доски** → вкладка **Макет карточки** → нажмите **Редактировать WIP-лимиты по полям**.
2. В модальном окне перечислены все существующие правила.
3. Чтобы **редактировать** правило: измените его поля (значение лимита, метку, цвета, область действия) на месте, затем нажмите **Сохранить**.
4. Чтобы **удалить** правило: нажмите иконку **Удалить** (корзина) рядом с правилом. Подтвердите удаление.
5. Все изменения требуют нажатия **Сохранить** для применения.

### Настройка отображения: метка и цвет бейджа

**Цель:** Сделать бейдж лимита узнаваемым на доске.

1. В форме лимита по полю введите **Краткую метку** (например, «Критичные баги»).
2. При необходимости задайте **цвет бейджа** через палитру цветов или hex-значение.
3. Метка и цвет отображаются на бейдже в панели инструментов доски.
4. Нажмите **Сохранить** для применения.

### Настройка области действия по колонкам и дорожкам

**Цель:** Применить лимит только к определённым колонкам или дорожкам.

1. В форме лимита по полю найдите селекторы **колонок** и **дорожек**.
2. По умолчанию лимит применяется ко **всем колонкам** и **всем дорожкам**.
3. Снимите **Все колонки** / **Все дорожки** и выберите конкретные для сужения области действия.
4. В лимите учитываются только задачи в выбранной области действия.
5. Нажмите **Сохранить** для применения.

### Представление доски: бейджи на панели инструментов и затемнение карточек

**Цель:** Отслеживать ресурсные ограничения на доске.

1. Настроенные лимиты по полям отображаются в виде **бейджей** в строке над колонками доски.
2. Каждый бейдж показывает **текущее / лимит** в выбранных вами цветах.
3. Когда лимит **превышен**, карточки, учитываемые в этом лимите, получают **красное затемнение** (полупрозрачное наложение), чтобы перегруженные группы выделялись.
4. Счётчики обновляются в реальном времени при перемещении карточек между колонками.

## Семантика сопоставления

- **Поле имеет значение:** совпадает, если поле не пустое (любое значение вообще). Пустые/незаполненные поля исключаются.
- **Точное значение:** совпадает, если значение поля равно настроенному значению. Сопоставление **без учёта регистра** и игнорирует начальные/конечные пробелы. Например, настройка `Критичный` соответствует `критичный`, `КРИТИЧНЫЙ` и `  Критичный  `.
- **Любое из значений:** такое же сопоставление без учёта регистра и пробелов, как у Точного значения, применяется к каждому настроенному значению независимо. Задача совпадает, если её значение поля совпадает с любым из списка.
- **Точное значение** и **Любое из значений** работают как с текстовыми, так и с полями выбора (выпадающие списки, радиокнопки, метки). Для полей выбора Jira сопоставление выполняется по отображаемой метке варианта.

## Пограничные случаи

- **Несколько правил совпадают с одной карточкой:** Если несколько правил лимитов по полям совпадают с одной карточкой (например, одно правило по Приоритету и другое по Компонентам), карточка учитывается обоими правилами независимо. Карточка может быть затемнена красным от одного превышенного лимита и при этом находиться ниже другого.
- **Сумма чисел с нечисловыми значениями:** Если значение поля карточки нечисловое (текст, пустое или отсутствует) при использовании режима Сумма чисел, карточка **пропускается** для этого лимита, и на бейдже отображается иконка предупреждения. Итог лимита суммирует только карточки с корректными числовыми значениями.
- **Поле удалено из макета карточки:** Если поле удалено из макета карточки после создания правила, правило продолжает существовать в настройках, но больше не совпадает ни с одной карточкой — фактически бездействует, пока поле не будет возвращено в макет.
- **Пустое значение поля:** Для режима «Поле имеет значение» пустые/незаполненные поля не учитываются. Для «Точного значения» и «Любого из значений» пустое значение можно явно указать, введя пустую строку.

## Устранение неполадок

### Бейдж показывает 0, но я ожидаю, что задачи будут учтены

1. Проверьте, что поле присутствует в **макете карточки** (Настройки доски → Макет карточки). Правила оценивают только поля, видимые на карточках.
2. Проверьте **область действия по колонкам и дорожкам** — задачи вне выбранной области не учитываются.
3. Для режима «Точное значение»: убедитесь, что значение совпадает (сопоставление без учёта регистра, но лишние пробелы или различия в пунктуации всё ещё имеют значение).
4. Для режима «Любое из значений»: проверьте каждое значение по отдельности.

### Правило не отображается на доске

1. Убедитесь, что вы нажали **Сохранить** после добавления или редактирования правил.
2. Перезагрузите страницу доски после сохранения — изменения применяются при следующей загрузке страницы.
3. Проверьте, что вы находитесь в детальном представлении доски (не Бэклог или Планирование).

### Счёт всегда 0

1. Для «Поле имеет значение»: поле должно быть непустым на совпадающих задачах.
2. Для «Сумма чисел»: поле должно содержать корректное число, не текст и не пустое. Карточки с нечисловыми значениями пропускаются и вызывают иконку предупреждения на бейдже.
3. Поле может присутствовать в макете карточки, но быть пустым на всех видимых задачах.

### Иконка предупреждения появляется на бейдже суммы чисел

- Иконка предупреждения на бейдже означает, что одна или несколько карточек имеют нечисловые значения в суммируемом поле и были пропущены. Наведите курсор на бейдж, чтобы увидеть, сколько карточек пропущено. Исправьте значения поля на этих карточках (или выберите другое поле).
