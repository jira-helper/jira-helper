# Column Group WIP Limits (CONWIP)

## Обзор

Column Group WIP Limits позволяют объединить несколько колонок под общим лимитом незавершённой работы (**CONWIP** — Constant Work In Progress). Вместо установки лимита на каждую колонку вы задаёте единый лимит для группы колонок, и все задачи в этих колонках учитываются совместно.

**Почему это важно:** Индивидуальные лимиты колонок не работают, когда этап рабочего процесса естественным образом охватывает несколько колонок (например, «In Progress» → «Review» → «QA»). Группа CONWIP ограничивает общий объём работы на всём этапе — именно так большинство команд на практике управляют потоком.

**Где появляется:** Функция активируется на виде **доски** (Board). Настройка выполняется через вкладку **Настройки доски → Columns** в диалоге настроек расширения.

## Предварительные требования

- Права **администратора доски** на целевой Jira-доске (необходимы для сохранения конфигурации в свойства доски)
- Расширение Jira Helper [установлено](/docs/getting-started/installation) и включено
- Доска Scrum или Kanban с как минимум **двумя колонками**

## Ограничения

- Колонка может принадлежать **не более чем одной группе**. Нельзя поместить одну и ту же колонку в две разные группы.
- Максимальное количество групп колонок определяется количеством колонок на вашей доске (для одной группы нужна как минимум одна колонка).
- Настройки хранятся как **свойства доски** — все участники команды с доступом к доске видят одни и те же группы и лимиты.

## Пользовательские сценарии

### Создание группы колонок

**Цель:** Сгруппировать несколько колонок и задать их общий WIP-лимит.

1. Откройте **Настройки доски** → вкладка **Columns**.
2. Нажмите **Column group WIP limits**, чтобы открыть диалог настройки.
3. Перетащите колонки в группу. Каждая группа получает имя по умолчанию, которое можно переименовать.
4. Установите число **Max issues** для группы.
5. Опционально задайте **пользовательский hex-цвет** для заголовка группы (см. [Установка цвета заголовка группы](#установка-цвета-заголовка-группы)).
6. Нажмите **Save** для сохранения конфигурации. Доска обновляется мгновенно.

### Редактирование группы колонок

**Цель:** Изменить имя, лимит, состав колонок или фильтры группы.

1. Откройте диалог настройки групп колонок (Настройки доски → Columns → Column group WIP limits).
2. Измените любую группу: переименуйте, измените значение **Max issues**, перетащите колонки внутрь или наружу, настройте фильтры swimlane/типов задач.
3. Нажмите **Save**. Доска отражает изменения мгновенно.

### Удаление группы колонок

**Цель:** Удалить группу и освободить её колонки.

1. В диалоге настройки групп колонок найдите группу для удаления.
2. Нажмите **значок удаления** (корзина/X) на заголовке группы.
3. Колонки группы возвращаются в пул несгруппированных.
4. Нажмите **Save** для завершения.

<!-- SCREENSHOT: Configuration dialog with a delete icon (X) highlighted on a group header, and columns flowing back to the available pool -->

### Настройка ограничения по swimlane

**Цель:** Выбрать, какие swimlane учитываются в лимите группы.

1. В диалоге настройки групп колонок нажмите фильтр **swimlane** группы.
2. Выберите **All swimlanes** (по умолчанию) для учёта задач из всех swimlane или выберите конкретные swimlane.
3. Только задачи в выбранных swimlane учитываются в лимите.
4. Нажмите **Save** для применения.

### Фильтрация по типам задач

**Цель:** Учитывать только определённые типы задач в лимите группы.

1. В диалоге групп колонок нажмите фильтр **типа задач** для группы.
2. Выключите **Count all issue types**.
3. Выберите **проект** и укажите конкретные типы задач (Bug, Story, Task и т.д.).
4. Учитываются только совпадающие типы задач. Остальные задачи в колонках группы игнорируются.
5. Нажмите **Save** для применения.

### Установка цвета заголовка группы

**Цель:** Задать отличительный цвет заголовка группы колонок для быстрой визуальной идентификации.

1. В диалоге групп колонок нажмите **палитру цветов** для группы.
2. Введите **hex-значение цвета** (например, `#ff5630`) или используйте палитру.
3. Заголовок колонки группы отображается с этим цветом на доске.
4. Если цвет не задан, он определяется автоматически на основе имени группы.

### Вид доски: мониторинг

**Цель:** Видеть заполненность группы и состояние перегрузки на доске.

1. На доске **первая колонка** каждой группы отображает значок **текущее / лимит** на своём заголовке (например, `7 / 10`).
2. Заголовок группы колонок имеет цветной фон, соответствующий назначенному цвету группы.
3. Когда группа **превышает** лимит, область колонки получает **красный фон**, сигнализируя о перегрузке.
4. Фильтры swimlane и типов задач учитываются — не все видимые задачи обязательно учитываются в значке.

## Пошаговый пример

**Сценарий:** Рабочий процесс вашей команды имеет три колонки для этапа разработки: «In Progress», «Code Review» и «QA». Вы хотите установить общий лимит в 8 задач на все три колонки.

1. Откройте доску → нажмите **Настройки доски** → вкладка **Columns** → **Column group WIP limits**
2. Перетащите «In Progress», «Code Review» и «QA» в новую группу
3. Переименуйте группу в **Development**
4. Установите **Max issues** = `8`
5. Опционально задайте цвет группы `#0052cc` (синий)
6. Нажмите **Save**

Результат: Заголовок колонки «In Progress» теперь показывает, например, `6 / 8`. Когда команда переносит 9-ю задачу в любую из трёх колонок, группа становится красной, сигнализируя о перегрузке.

## Устранение неполадок

### Счётчик не совпадает с видимым количеством карточек

1. Проверьте **фильтр swimlane** группы — задачи в исключённых swimlane не учитываются
2. Проверьте **фильтр типов задач** группы — только выбранные типы задач учитываются в счётчике
3. Счётчик учитывает задачи во **всех колонках** группы, а не только в той, где отображается значок
4. Подзадачи могут быть исключены в зависимости от вашей конфигурации фильтров

### Цвет/счётчик группы не отображаются на доске

1. **Обновите** страницу доски (F5) — DOM может требовать перезагрузки после сохранения настроек
2. Проверьте, что в группе есть хотя бы **одна колонка** (пустые группы не отображаются)
3. Убедитесь, что расширение **включено** и вы находитесь на **виде доски** (не в бэклоге)
4. Если колонки были переименованы в Jira после создания группы, старые ссылки на колонки могут быть нарушены — удалите и создайте группу заново

### Настройки не сохраняются / кнопка «Save» не работает

1. Убедитесь, что у вас есть права **администратора доски** (запись в свойства доски завершается без ошибок при отсутствии прав администратора)
2. Проверьте, что у всех групп задано значение **Max issues** (пустой лимит = недействительная группа)
3. Попробуйте обновить страницу и снова открыть диалог настроек

### Колонка уже принадлежит другой группе

Каждая колонка может принадлежать **не более чем одной группе**. Если вы видите серую колонку в доступном пуле, она уже в группе. Сначала удалите её из текущей группы, затем перетащите в новую.
