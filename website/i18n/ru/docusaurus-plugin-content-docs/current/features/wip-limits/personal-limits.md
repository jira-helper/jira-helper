# Персональные WIP-лимиты

## Обзор

Персональные WIP-лимиты контролируют, сколько задач может быть у каждого участника команды в работе одновременно. На доске у каждого пользователя отображаются аватар-бейджи, показывающие текущую загрузку относительно лимита, с цветовой индикацией и опциональной подсветкой карточек при превышении лимита.

Используйте это для балансировки загрузки в команде, выявления перегруза до стендапов и фокусировки на задачах одного человека одним кликом. Функция доступна на представлении **Доска** и настраивается в **Настройки доски → вкладка Колонки**.

**Необходимые условия:**
- Для настройки лимитов требуются права **администратора доски**
- У каждого участника команды должна быть установлена **фотография аватара в Jira** (силуэт по умолчанию не отображается в виде бейджа)
- Участник должен быть пользователем Jira, видимым в селекторе пользователей доски

## Пользовательские сценарии

### Добавить лимит для пользователя

**Цель:** Установить ограничение WIP для конкретного участника команды.

1. Откройте **Настройки доски** → вкладка **Колонки**.
2. Нажмите **Управление персональными WIP-лимитами**.
3. В модальном окне используйте поле **поиска пользователя** — начните вводить имя и выберите пользователя из выпадающего списка.
4. Установите число **Максимум задач в работе** (минимум 1).
5. Нажмите **Добавить лимит**, затем нажмите **Сохранить**, чтобы применить для всех пользователей доски.
6. Аватар-бейдж появляется рядом с заголовком доски сразу после сохранения.

### Настройка области действия: колонки, дорожки, типы задач

**Цель:** Ограничить лимит конкретными колонками, дорожками или типами задач.

1. В форме лимита найдите фильтры области действия:
   - **Колонки** — отключите **Все колонки** и выберите конкретные колонки.
   - **Дорожки** — отключите **Все дорожки** и выберите конкретные дорожки.
   - **Типы задач** — отключите **Учитывать все типы задач**, выберите проект, затем выберите типы.
2. В лимите учитываются только задачи, соответствующие всем активным фильтрам.
3. Допускается несколько лимитов для одного пользователя с разными областями действия (например, 3 задачи в «В работе» + 2 бага везде).
4. Нажмите **Сохранить** для применения.

### Создание общих лимитов (несколько человек делят один счётчик)

**Цель:** Один WIP-счётчик на нескольких человек — каждый пользователь видит одинаковый общий итог на своём бейдже.

**Как работают общие лимиты:** Когда вы настраиваете несколько человек с одинаковой областью действия и значением лимита, все их назначенные задачи учитываются в едином общем счётчике. На бейдже аватара каждого пользователя отображается **одинаковый общий счёт** относительно **одного и того же общего лимита**, а не индивидуальные значения.

1. В модальном окне персональных лимитов добавьте лимит для пользователя А с определённой областью действия и значением лимита.
2. Добавьте ещё один лимит для пользователя Б с **идентичной областью действия** и **тем же значением лимита**.
3. Два лимита автоматически распознаются как группа общего лимита — все задачи, назначенные любому пользователю в группе, учитываются в едином общем счётчике.
4. Бейдж каждого пользователя показывает `общий_счёт / общий_лимит` (например, у обоих аватаров отображается `7 / 10`).
5. Нажмите **Сохранить** для применения.

### Фильтрация доски по пользователю

**Цель:** Показать только задачи, учитываемые в лимите конкретного пользователя.

1. На доске нажмите на **аватар-бейдж** пользователя.
2. Доска фильтруется, показывая только задачи, попадающие в область действия этого лимита.
3. Все остальные карточки скрыты.
4. Нажмите на тот же аватар-бейдж ещё раз, чтобы сбросить фильтр и показать все карточки.
5. Если у пользователя несколько лимитов, у каждого лимита свой аватар-бейдж с независимой фильтрацией.

### Представление доски: аватар-бейджи и подсветка карточек

**Цель:** Отслеживать загрузку команды с первого взгляда.

1. Настроенные лимиты отображаются в виде **аватар-бейджей** (фото профиля Jira + счёт) в области панели инструментов доски.
2. Каждый бейдж показывает **текущее / лимит** (например, `2 / 5`).
3. Цветовая кодировка:
   - **Зелёный** — ниже лимита
   - **Жёлтый** — на пределе лимита
   - **Красный** — превышение лимита, карточки, учитываемые в этом лимите, выделяются красным фоном
4. Учитываются задачи, назначенные пользователю и попадающие в область действия лимита. Для отображения требуется аватар Jira.

## Устранение неполадок

### Бейджи не отображаются для некоторых пользователей

1. **Нет фото аватара в Jira:** Если у пользователя силуэт по умолчанию вместо фото, бейдж не отображается. Попросите пользователя загрузить фото профиля в настройках профиля Jira.
2. **Пользователь не найден:** Убедитесь, что пользователь существует в Jira и виден в селекторе пользователей доски. Деактивированные пользователи не отображаются.
3. **Нет подходящих задач:** Бейдж появляется, только если есть задачи, назначенные пользователю в пределах области действия лимита. Если все его задачи находятся в исключённых колонках или отфильтрованы по типу задачи, бейдж показывает `0 / лимит` — но может отображаться мелким.
4. **Совместимость с представлением доски:** Бейджи отображаются только в детальном представлении доски (`?view=detail`). Они не отображаются в представлениях Бэклог, Планирование или Упрощённое.

### Счёт на бейдже не соответствует ожидаемому числу

1. **Фильтры области действия:** Проверьте фильтры колонок, дорожек и типов задач для лимита. Учитываются только задачи, соответствующие всем трём категориям фильтров.
2. **Поле исполнителя:** Счёт использует поле Исполнитель в Jira. Убедитесь, что задача действительно назначена этому пользователю (не неназначена и не назначена другому).
3. **Поведение общего лимита:** Если лимит общий, на бейдже каждого пользователя отображается общий итог по всей группе, а не его индивидуальный счёт. Это сделано намеренно.

### Не удаётся найти пользователя в выпадающем списке поиска

1. Пользователь должен быть пользователем Jira с доступом к проекту доски.
2. Селектор пользователей показывает только тех, кто хотя бы раз просматривал этот экземпляр Jira.
3. Введите не менее 2 символов для запуска поиска.
