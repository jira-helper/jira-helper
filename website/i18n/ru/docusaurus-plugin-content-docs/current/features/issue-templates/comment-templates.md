# Шаблоны комментариев

## Обзор

Шаблоны комментариев позволяют сохранять, управлять и повторно использовать часто используемый текст комментариев прямо в редакторе задач Jira. Вместо повторного набора одних и тех же обновлений статуса, заметок о встречах или сообщений о передаче задач вы вставляете заранее подготовленный шаблон одним кликом из панели инструментов редактора. Шаблоны поддерживают пресеты наблюдателей (автоматическое добавление коллег), индивидуальные цвета для визуальной группировки и импорт/экспорт JSON для обмена с командой.

**Зачем:** Команды, следующие структурированным шаблонам общения (ежедневные обновления стендапа, заметки о релизах, чек-листы код-ревью), тратят время на копирование и вставку одного и того же текста. Шаблоны комментариев сводят эти трудозатраты к одному клику.

**Где:** Кнопка **Шаблоны** появляется на панели инструментов редактора комментариев в каждой задаче Jira. Шаблоны хранятся в `localStorage` вашего браузера — персональные, не передаются через Jira.

## Необходимые условия

- Права на доску не требуются — шаблоны хранятся в `localStorage` (см. [Настройки: Локальное хранилище](/docs/settings#local-storage-personal))
- Расширение Jira Helper должно быть активно на странице
- Редактор комментариев должен быть открыт (нажмите **Комментарий** на любой задаче)

## Пользовательские сценарии

### Управление шаблонами: добавление, редактирование, удаление

**Цель:** Создать и поддерживать библиотеку многоразовых шаблонов комментариев.

1. Откройте задачу Jira → нажмите кнопку **Шаблоны** на панели инструментов редактора комментариев.
2. Выберите **Управление шаблонами**, чтобы открыть панель настроек.
3. **Добавить шаблон:** Заполните **Метку** (отображается в выпадающем списке), **Текст** (тело комментария — поддерживает обычный текст и подмножество wiki-разметки Jira, см. ниже), опционально добавьте **Наблюдателей** (поиск и выбор пользователей Jira по имени) и выберите **Цвет** для визуальной группировки.
4. **Редактировать шаблон:** Кликните по существующей строке шаблона и измените любое поле.
5. **Удалить шаблон:** Нажмите значок удаления (корзина) рядом со строкой шаблона. Подтвердите удаление.
6. Шаблоны хранятся в `localStorage` для домена Jira. Черновые изменения обозначаются баннером **Несохранённые изменения**.
7. Нажмите **Сохранить**, чтобы сохранить все черновые изменения, или **Отменить**, чтобы вернуться к исходному состоянию.

<!-- SCREENSHOT: Manage templates panel — list of 4 templates with colored labels, one template expanded showing Text field with wiki markup, Watchers field with 2 selected users, Color picker open -->

**Поддерживаемая wiki-разметка:**

- **Форматирование текста:** `*жирный*`, `_курсив_`, `+подчёркнутый+`, `-зачёркнутый-`
- **Заголовки:** `h1.`, `h2.`, `h3.`, `h4.`
- **Списки:** `* ` для маркированных, `# ` для нумерованных
- **Ссылки:** `[текст|url]`
- **Код:** `{{встроенный код}}`, `{code}...{code}` блок
- **Цитаты:** `{quote}...{quote}`
- **Горизонтальная линия:** `----`
- **Разрывы строк:** пустая строка разделяет абзацы

**Не поддерживается:** макросы `{panel}`, блоки `{noformat}`, изображения, вложения, упоминания (`[~username]`), таблицы и другие макросы плагинов Jira. HTML-теги удаляются.

### Импорт/экспорт шаблонов в JSON

**Цель:** Переносить шаблоны между браузерами, делиться с коллегами или создавать резервную копию библиотеки.

1. Откройте **Управление шаблонами**.
2. **Экспорт:** Нажмите **Экспорт шаблонов**, чтобы скачать JSON-файл со всеми текущими шаблонами.
3. **Импорт:** Нажмите **Импорт JSON-файла**, выберите файл `.json`. Если формат файла неверен, показывается уведомление **Ошибка импорта** с подробностями.
4. Импортированные шаблоны добавляются как черновые изменения — просмотрите список и нажмите **Сохранить** для сохранения.

**Структура JSON импорта:**

```json
[
  {
    "label": "Обновление стендапа",
    "text": "*Вчера:* ...\n*Сегодня:* ...\n*Блокировки:* ...",
    "watchers": ["user1", "user2"],
    "color": "#1890ff"
  }
]
```

- `label` (обязательно, строка) — отображаемое имя в выпадающем списке
- `text` (обязательно, строка) — тело комментария (поддерживается wiki-разметка Jira, см. список поддерживаемой разметки выше)
- `watchers` (опционально, массив строк) — массив имён пользователей Jira (не отображаемых имён)
- `color` (опционально, hex-строка) — CSS-цвет для записи шаблона

### Вставка шаблона из панели инструментов редактора

**Цель:** Вставить заранее подготовленный комментарий одним кликом.

1. При редактировании комментария к любой задаче найдите выпадающий список **Шаблоны** на панели инструментов редактора.
2. Кликните по названию шаблона — его содержимое вставляется в позицию курсора.
3. Если у шаблона настроены **Наблюдатели**, расширение пытается добавить их после вставки. Статус показывается в уведомлении:
   - *Наблюдатели добавлены* — все успешно добавлены
   - *Некоторые наблюдатели не добавлены* — частичный успех (наведите курсор, чтобы увидеть, какие не удалось добавить)
   - *Наблюдатели не добавлены* — все не удалось (вероятно, из-за ограничений прав или неверных имён пользователей)
4. Если ключ задачи недоступен (например, редактор на странице, не являющейся задачей), шаблон вставляется, но наблюдатели пропускаются с уведомлением.

## Ограничения

- **Только локально:** Шаблоны хранятся в `localStorage`. Они **не** синхронизируются между браузерами или устройствами. Используйте Экспорт/Импорт для переноса между машинами.
- **Максимальное количество:** В выпадающем списке отображается до 50 шаблонов. Свыше 50 навигация по списку становится затруднительной. Рекомендуется экспортировать и архивировать неиспользуемые шаблоны, а не держать их в выпадающем списке.
- **Нет кросс-браузерной синхронизации:** Каждый профиль браузера имеет собственное независимое хранилище шаблонов.
- **Нет форматированного текста:** Содержимое шаблона — это обычный текст с подмножеством wiki-разметки Jira, перечисленным выше. HTML, изображения и вложения не поддерживаются.
- **Права на наблюдателей:** Добавление наблюдателей требует права проекта **Manage Watchers** в Jira. Если у вас нет этого права, показывается уведомление о невозможности добавления наблюдателей.

## Устранение неполадок

### Кнопка «Шаблоны» отсутствует на панели инструментов редактора

- Убедитесь, что расширение Jira Helper включено и активно на странице
- Редактор комментариев должен быть **в фокусе** или **заполнен**, чтобы панель инструментов полностью отобразилась — сначала кликните в поле редактора
- На очень старых версиях Jira Server структура DOM панели инструментов может отличаться — обновите расширение до последней версии

### Импорт завершается с «Ошибкой импорта»

- Проверьте, что JSON-файл корректен — откройте его в текстовом редакторе и проверьте на синтаксические ошибки (висячие запятые, ключи без кавычек)
- Убедитесь, что файл содержит **массив** объектов шаблонов, а не одиночный объект: `[...]` а не `{...}`
- Обязательные поля: каждый объект должен иметь свойства `label` и `text`
- Имена пользователей Jira в `watchers` должны быть точными (с учётом регистра). Используйте выбор пользователя в Jira для проверки имён

### Шаблоны потеряны после очистки данных браузера

- Шаблоны хранятся в `localStorage`. Если вы или ваша IT-политика очищаете `localStorage` (например, при закрытии браузера или через управление конечными точками), все шаблоны удаляются
- **Предотвращение потери данных:** регулярно экспортируйте шаблоны через **Экспорт шаблонов** и храните JSON-файл в надёжном месте

### Наблюдатели не добавлены после вставки шаблона

- Проверьте, что у вас есть право проекта **Manage Watchers** в Jira для целевого проекта
- Имена пользователей в шаблонах должны соответствовать точному формату имён Jira (не отображаемым именам). Используйте выбор пользователя в **Управлении шаблонами** для добавления пользователей — он запрашивает API Jira для получения точных имён
- Если добавление наблюдателей не удалось, в редакторе комментариев появляется уведомление. Проверьте уведомление для получения сведений о том, какие пользователи не были добавлены.
