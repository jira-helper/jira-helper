# Шаблоны комментариев

| | |
|---|---|
| Где настраивается | Board Settings → вкладка Comment templates или Issue Settings → вкладка Comment templates |
| Где видно | Страница задачи (тулбар редактора комментариев) |
| Настройки действуют | Только для вас |(`jira_helper_comment_templates`) |

## Цель

Предоставить переиспользуемые шаблоны комментариев, которые вставляются одним кликом из тулбара редактора комментариев, с опциональным автоматическим добавлением наблюдателей.

## Как настроить

1. Откройте настройки доски через **Jira Helper** рядом с боковой панелью или настройки задачи через кнопку **Helper** в тулбаре задачи и выберите вкладку **Comment templates**.
2. Нажмите **Add template**, чтобы создать новый шаблон. Каждый шаблон содержит:
   - **Label** — короткое название, отображаемое на кнопке тулбара.
   - **Color** — предустановленный или свой hex-цвет для кнопки.
   - **Text** — текст комментария для вставки.
   - **Watchers** (опционально) — пользователи Jira, добавляемые в наблюдатели при вставке шаблона.
3. Редактируйте, переупорядочивайте или удаляйте шаблоны по необходимости.
4. Нажмите **Save**, чтобы сохранить изменения в локальном хранилище браузера.
5. Используйте **Import JSON file** и **Export templates** для переноса шаблонов между браузерами.
6. Используйте **Reset to defaults** для восстановления встроенного набора шаблонов.

Шаблоны персональные и хранятся в вашем браузере, а не на сервере Jira.

## Как использовать

- В **редакторе комментариев** на любой странице задачи появляется тулбар с подписанными цветными кнопками шаблонов.
- Нажмите кнопку шаблона, чтобы **вставить** его текст в поле комментария.
- Если в шаблоне настроены наблюдатели, они автоматически добавляются к задаче.
- Кратковременное уведомление подтверждает успех вставки или сообщает об ошибках добавления наблюдателей.
- Нажмите **Manage templates** в тулбаре для открытия панели настроек.

## Сценарии использования

- **Стандартные ответы:** Создавайте шаблоны для типовых ответов (например, «Буду разбираться», «Слито в master», «Нужно ревью»).
- **Онбординг:** Делитесь JSON-экспортом с новыми членами команды, чтобы у них были те же шаблоны.
- **Автоматизация наблюдателей:** Включайте наблюдателей в шаблоны для автоматического оповещения нужных людей.
- **Многоязычные команды:** Создавайте шаблоны на разных языках для кросс-командной коммуникации.
