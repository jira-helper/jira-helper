# Отображение связей задач

## Обзор

Отображение связей задач показывает связанные задачи в виде цветных бейджей непосредственно на карточках доски. Каждый бейдж отображает сводку связанной задачи, а при нажатии открывает связанную задачу в новой вкладке. Вы определяете, какие типы связей показывать, какие исходные задачи проверять и какие связанные задачи фильтровать — это даёт полный контроль над тем, какие отношения видны на доске.

**Зачем:** Понимание зависимостей и отношений между задачами обычно требует открытия каждой карточки. Встроенные бейджи связей делают контекст видимым с первого взгляда — родительские задачи, блокировки, связанные проекты и многое другое — не покидая доску.

**Где:** Настройки доски → Jira Helper → Дополнительные элементы карточек → Отображение связей задач на карточках. Конфигурации управляют тем, какие связи отображаются, с фильтрацией по типу связи, исходным фильтрам и фильтрам связанных задач.

**Как это работает:**
- Вы определяете **конфигурации связей**: каждая имеет имя, тип связи с направлением, опциональные исходные фильтры, опциональные фильтры связанных задач, настройки цвета и переключатель многострочности
- Конфигурации могут фильтровать, какие исходные задачи анализировать (все задачи, или по JQL / значениям полей)
- Конфигурации могут фильтровать, какие связанные задачи отображать (все связанные, или по JQL / значениям полей)
- На доске: бейджи отображаются под заголовком карточки, вертикально в выбранных колонках
- В Бэклоге: бейджи отображаются горизонтально в ряд, когда включено `Показывать связи в бэклоге`
- Каждый бейдж показывает сводку связанной задачи; нажмите на него, чтобы открыть задачу в новой вкладке
- Цвета могут быть фиксированными или автоматически генерируемыми уникальными для каждой связанной задачи

## Необходимые условия

- Для создания или изменения конфигураций связей требуются **права редактирования доски** (настройки хранятся в [свойствах доски](/docs/settings#board-properties-team-shared))
- В экземпляре Jira должны быть определены типы связей задач (встроенные типы, такие как «блокирует», «связана с», всегда доступны)

---

## Быстрый старт: Предустановки распространённых типов связей

| Тип связи (Входящая) | Тип связи (Исходящая) | Сценарий использования |
|---|---|---|
| `заблокирована` | `блокирует` | Показать, что блокирует каждую карточку |
| `клонирована` | `клонирует` | Отслеживать связи клонирования |
| `связана с` | `связана с` | Показать слабо связанную работу |
| `является родителем` | `является дочерней` | Показать иерархию родитель/потомок |
| `вызвана` | `вызывает` | Отслеживать причинно-следственные цепочки |
| `дублирована` | `дублирует` | Показать задачи-дубликаты |

> **Совет:** Используйте **Уникальные цвета** для конфигураций, отслеживающих несколько целей (например, блокировки), чтобы каждый блокирующий элемент получал отличительный цвет бейджа. Используйте **Фиксированные цвета** для отношений родитель/потомок, где единообразие помогает.

## Пользовательские сценарии

### Добавить конфигурацию связи: имя, тип связи, направление

**Цель:** Создать новую конфигурацию связи для отображения определённого типа отношений.

1. Откройте настройки доски → **Jira Helper** → вкладка **Дополнительные элементы карточек**.
2. Убедитесь, что **Включить дополнительные элементы карточек** отмечено.
3. Нажмите **Добавить конфигурацию связи**.
4. В новой карточке конфигурации введите **Имя связи** (макс. 20 символов, отображается только в настройках).
5. Выберите **Тип связи** из выпадающего списка. Список автоматически загружается из вашего экземпляра Jira. Примеры: `является родителем`, `связана с`, `блокирует`, `заблокирована`.
6. Направление (входящее/исходящее) определяется выбранным типом связи.

### Настройка исходных фильтров (JQL или поле на исходных задачах)

**Цель:** Ограничить, какие задачи на доске проверяются на наличие связей.

1. В конфигурации связи снимите флажок **Отслеживать все задачи**.
2. Доступны два режима фильтрации:
   - **Режим JQL:** введите JQL-запрос (например, `status = "In Progress"`)
   - **Режим поля:** выберите ID поля и значение (например, Тип задачи = Проект)
3. Только задачи, соответствующие фильтру, будут проанализированы на наличие связей и отображены.

### Настройка фильтров связанных задач (JQL или поле)

**Цель:** Ограничить, какие связанные задачи отображаются в виде бейджей.

1. В конфигурации связи снимите флажок **Отслеживать все связанные задачи**.
2. Настройте фильтр в режиме JQL или поля:
   - **Режим JQL:** введите JQL-запрос (например, `status != "Done"`)
   - **Режим поля:** выберите поле и значение (например, Тип задачи = Проект)
3. Только связанные задачи, соответствующие фильтру, отображаются в виде бейджей.

### Настройка отображения (цвет, многострочная сводка)

**Цель:** Управлять визуальным видом бейджей связей.

1. В конфигурации связи найдите флажок **Уникальные цвета**:
   - **Включено:** каждая связанная задача получает автоматически сгенерированный уникальный цвет на основе её ключа и сводки
   - **Отключено:** все бейджи используют один фиксированный цвет, выбираемый через палитру цветов
2. Найдите флажок **Многострочная сводка**:
   - **Включено:** длинные сводки переносятся на несколько строк
   - **Отключено:** длинные сводки обрезаются с многоточием; полный текст появляется при наведении

### Включение в представлении Бэклога

**Цель:** Показывать бейджи связей также на карточках Бэклога.

1. В настройках **Дополнительные элементы карточек** найдите флажок **Показывать связи в бэклоге**.
2. Установите флажок.
3. Карточки Бэклога теперь отображают бейджи связей в горизонтальном ряду в конце карточки.
4. Настройки колонок не применяются к Бэклогу — связи показываются для всех задач Бэклога.

### Сделать связи эпиков кликабельными

**Цель:** Открывать родительский эпик непосредственно из стандартного бейджа Epic Link на карточке.

1. В настройках **Дополнительные элементы карточек** найдите флажок **Сделать Epic Link кликабельным**.
2. Установите флажок.
3. Стандартные бейджи Epic Link на карточках становятся кликабельными. При нажатии эпик открывается в новой вкладке.

### Представление доски: цветные бейджи под заголовком карточки

**Цель:** Читать связанные задачи непосредственно на карточках доски.

1. Посмотрите на карточки в выбранных колонках. Под сводкой/заголовком каждой карточки вертикально отображаются цветные бейджи (по одному на связанную задачу).
2. Каждый бейдж показывает текст сводки связанной задачи.
3. Цвет бейджа зависит от конфигурации: фиксированный цвет или уникальный автоматически сгенерированный цвет для каждой задачи.
4. **Наведите курсор** на бейдж, чтобы увидеть полную сводку, если она обрезана.
5. **Нажмите** на бейдж, чтобы открыть связанную задачу в новой вкладке браузера.
6. Несколько конфигураций накладываются — бейджи каждой конфигурации отображаются вместе под заголовком карточки.

## Пример конфигурации

**Цель:** Показать блокировки и связанную работу на инженерной доске.

| Настройка | Значение |
|---|---|
| **Имя связи** | Блокировки |
| **Тип связи** | `заблокирована` |
| **Отслеживать все задачи** | не отмечено |
| **Исходный JQL** | `status != "Done"` |
| **Отслеживать все связанные задачи** | не отмечено |
| **JQL связанных задач** | `status != "Done" AND status != "Cancelled"` |
| **Уникальные цвета** | включено |
| **Многострочная сводка** | включено |

С этой конфигурацией карточки в активных колонках отображают цветные бейджи для каждой неразрешённой блокировки. Завершённые карточки и отменённые блокировки скрыты для уменьшения шума.

## Соображения по производительности

- Каждая активная конфигурация запускает API-запрос к Jira для каждой соответствующей карточки. На досках с **50+ карточками и 3+ конфигурациями связей** отрисовка бейджей может задерживаться на 1–3 секунды при загрузке доски или смене колонок.
- Ограничьте количество активных конфигураций теми, которые вы действительно используете. Отключайте неиспользуемые конфигурации, а не удаляйте их.
- Используйте исходные фильтры и фильтры связанных задач для уменьшения количества задач, запрашиваемых на конфигурацию.

## Устранение неполадок

### Бейджи не отображаются на карточках

- Проверьте, что **Включить дополнительные элементы карточек** отмечено
- Убедитесь, что **Настройки колонок** включают колонку, в которой находится карточка
- Проверьте, что тип связи существует в вашем экземпляре Jira (выпадающий список автоматически загружается из Jira — если пуст, в Jira может не быть пользовательских типов связей)
- Если используются исходные фильтры, проверьте, компилируется ли JQL: протестируйте его непосредственно в Jira на странице **Задачи → Поиск задач**

### Связи эпиков не кликабельны

- Убедитесь, что **Сделать Epic Link кликабельным** отмечено
- Бейджи Epic Link появляются только когда тип задачи карточки включён в схему Epic Link (стандартно в Scrum-досках Jira)
- Функция работает только на досках Jira Software, не в Jira Service Management

### Связи кажутся устаревшими / закэшированными

- Данные связей запрашиваются при загрузке доски и когда Jira инициирует обновления дорожек/карточек. Если вы редактировали связи в другой вкладке, **обновите страницу доски** (F5) для принудительной перезагрузки.
- Если бейджи всё ещё показывают устаревшие данные, проверьте консоль браузера на наличие ошибок API — экземпляр Jira может ограничивать частоту запросов

### Ошибки синтаксиса JQL в фильтрах

- Тестируйте все JQL-запросы непосредственно в навигаторе задач Jira перед вставкой в конфигурацию
- Частые ошибки: имена статусов с пробелами без кавычек (`status = In Progress` → `status = "In Progress"`), неправильные имена полей (`assignee` → `assignee` правильно; `assignedTo` — нет)
