# Проверки условий задач

## Обзор

Проверки условий задач размещают пользовательские иконки-бейджи на карточках доски, когда задача (или связанная работа) соответствует условию JQL, которое вы определяете. В отличие от цветов карточек Jira, которые применяют полный фон карточки по правилам, это маленькие встроенные иконки, которые могут содержать текст подсказки, анимацию и цвет — идеально для выделения специфических свойств, которые Jira не отображает визуально на карточках.

**Зачем:** Стандартные доски Jira не позволяют добавлять пользовательские визуальные сигналы по произвольным условиям JQL. С помощью Проверок условий вы можете помечать задачи, у которых есть блокировки, отсутствуют оценки, которые принадлежат определённому релизу или соответствуют любому запрашиваемому условию — всё в виде видимых иконок на карточке.

**Где:** Настройки доски → Jira Helper → Дополнительные элементы карточек → Проверки условий задач.

**Как это работает:**
- Каждая проверка условия имеет: имя, JQL-запрос, иконку (30+ вариантов), цвет (опциональный фон), текст подсказки, анимацию (опционально)
- Два режима:
  - **Простой:** условие выполняется, когда сама карточка удовлетворяет JQL
  - **С учётом подзадач:** условие проверяет карточку + её подзадачи, дочерние элементы эпика или связанные задачи
- Режим совпадения подзадач: `любая` (хотя бы одна дочерняя совпадает) или `все` (каждая дочерняя совпадает)
  - Когда у карточки **ноль подзадач** и режим `все`, условие *не* совпадает (формальная истина на пустом множестве не применяется — должна быть хотя бы одна подзадача)
- Иконки поддерживают анимации: pulse, breathe, blink, blinkFast, shake
- Источники подзадач: прямые подзадачи, дочерние элементы эпика, связанные задачи (каждый можно переключить)
- Несколько условий могут быть активны одновременно — все совпадающие иконки отображаются на карточке

## Необходимые условия

- **Права редактирования доски** для создания или изменения проверок условий
- Знакомство с [синтаксисом JQL](/docs/advanced/jql-reference) — каждое условие использует JQL-запрос

---

## Популярные рецепты JQL

| Сценарий | JQL | Режим | Рекомендуемая иконка |
|---|---|---|---|
| Заблокированные задачи | `status = Blocked` | Простой | ⚠️ warning |
| Отсутствуют story points | `"Story Points" is EMPTY` | Простой | 📋 clipboard |
| Отсутствуют оценки | `timeestimate is EMPTY` | Простой | ⏱️ clock |
| Помеченные для релиза | `fixVersion in unreleasedVersions()` | Простой | 🚀 rocket |
| Просроченные | `duedate < now() AND resolution is EMPTY` | Простой | 🔴 fire |
| Есть неразрешённые блокировки | `issueLinkType = "is blocked by"` | Простой | 🔒 lock |
| Отмеченные (требуют внимания) | `Flagged = "Yes"` | Простой | 🚩 flag |
| Подзадача заблокирована (любая) | JQL задачи: _пусто_, JQL подзадачи: `status = Blocked`, режим: любая | С подзадачами | ⚠️ warning |
| Все подзадачи завершены | JQL задачи: _пусто_, JQL подзадачи: `status = Done`, режим: все | С подзадачами | ✅ check |

> Скопируйте любой JQL выше, вставьте его в проверку условия и сначала протестируйте в Навигаторе задач Jira, чтобы убедиться, что он возвращает ожидаемые задачи.

## Пользовательские сценарии

### Добавить условие: имя, JQL, иконка, цвет, подсказка

**Цель:** Создать новую проверку условия, которая отображает иконку на совпадающих карточках.

1. Откройте настройки доски → **Jira Helper** → вкладка **Дополнительные элементы карточек**.
2. Убедитесь, что **Включить дополнительные элементы карточек** отмечено.
3. Прокрутите до **Проверки условий задач** и нажмите **Добавить проверку условия**.
4. Заполните конфигурацию:
   - **Имя:** метка для этого условия (отображается только в настройках)
   - **JQL:** условие на языке запросов Jira для сопоставления (например, `status = Blocked` или `labels = urgent`)
   - **Иконка:** выберите из 30+ иконок (warning, bug, rocket, fire, lock, eye, bell, lightning и многие другие)
   - **Цвет:** выберите фоновый цвет для бейджа иконки (hex-формат) или оставьте неопределённым для отсутствия фона
   - **Подсказка:** текст, показываемый при наведении (например, «Эта задача заблокирована»)
5. Нажмите сохранить. Карточки, соответствующие JQL, теперь отображают выбранную иконку.

### Настройка учёта подзадач

**Цель:** Показывать иконку условия, когда подзадачи, дочерние элементы эпика или связанные задачи совпадают, а не (или в дополнение к) сама карточка.

1. В проверке условия измените режим с **Простой** на **С подзадачами**.
2. Задайте **JQL задачи** — условие, которому должна удовлетворять родительская карточка.
3. Задайте **JQL подзадачи** — условие, которому должны удовлетворять подзадачи/дочерние элементы.
4. Настройте **Режим совпадения подзадач:**
   - **Любая:** иконка появляется, если хотя бы одна подзадача совпадает
   - **Все:** иконка появляется, только если каждая подзадача совпадает (и существует хотя бы одна подзадача)
5. Настройте **Источники подзадач** (какую дочернюю работу проверять):
   - **Прямые подзадачи:** стандартные подзадачи Jira (по умолчанию: вкл)
   - **Дочерние элементы эпика:** задачи, связанные через поле Epic Link (по умолчанию: вкл)
   - **Связанные задачи:** задачи, связанные через связи задач Jira (по умолчанию: выкл)
6. Иконка теперь появляется только при выполнении условия на уровне подзадач.

### Выбор анимации

**Цель:** Добавить движение для привлечения внимания к совпадающим карточкам.

1. В проверке условия найдите выпадающий список **Анимация**.
2. Выберите из:
   - `none` — статичная иконка (по умолчанию)
   - `pulse` — мягкая пульсация
   - `breathe` — медленное появление/исчезновение
   - `blink` — мигание вкл/выкл
   - `blinkFast` — быстрое мигание
   - `shake` — короткая вибрация
3. Анимация выполняется непрерывно, пока условие совпадает.

### Представление доски: иконки-бейджи на совпадающих карточках

**Цель:** Видеть иконки проверок условий на карточках доски.

1. Посмотрите на карточки в выбранных колонках. Совпадающие карточки отображают иконки-бейджи в нижней части карточки.
2. Каждая активная проверка условия добавляет свою иконку. На одной карточке может отображаться несколько иконок.
3. **Наведите курсор** на иконку, чтобы увидеть текст подсказки, объясняющий условие.
4. Если карточка больше не соответствует JQL (например, статус изменился, метка удалена), иконка автоматически исчезает при обновлении доски.
5. Если настроены анимации, иконка анимируется на месте для привлечения внимания.

## Соображения по производительности

- Каждая активная проверка условия выполняет отдельный API-запрос к Jira для каждой видимой карточки. При **5+ активных условиях на доске с 50 карточками** загрузка может занять 2–4 секунды.
- Условия с учётом подзадач более затратны, чем простые условия — они запрашивают данные подзадач/дочерних элементов в дополнение к карточке. Ограничьте количество проверок с учётом подзадач самыми критичными сигналами (например, «есть заблокированная подзадача») и используйте простой режим где возможно.
- Если производительность доски снижается, отключите проверки условий, которые вы больше не используете, вместо того чтобы оставлять их активными.

## Устранение неполадок

### Иконка не отображается на карточке, которая должна совпадать

- Протестируйте JQL непосредственно в Jira на странице **Задачи → Поиск задач**. Если там ноль результатов, на доске совпадений тоже не будет.
- Для режима подзадач **все**: у карточки должна быть хотя бы одна подзадача — карточка с нулём подзадач никогда не совпадает в режиме `все`.
- Проверьте, что колонка карточки выбрана в **Настройках колонок** в разделе Дополнительные элементы карточек.
- Если используются источники подзадач, убедитесь, что переключатели источников включены. **Связанные задачи** по умолчанию выключены.

### JQL-запрос возвращает ошибки

- Значения с пробелами без кавычек вызывают ошибки разбора: `status = In Progress` → `status = "In Progress"`
- Имена полей чувствительны к регистру: `story points is EMPTY` → `"Story Points" is EMPTY`
- Сначала протестируйте в Навигаторе задач Jira — сообщения об ошибках там более подробные, чем в расширении

### Анимация не видна

- Анимации используют CSS; они могут быть отключены на уровне ОС (Windows: «Отключить все ненужные анимации» в настройках Специальных возможностей)
- Некоторые анимации (`blinkFast`, `shake`) намеренно агрессивны — если они кажутся сломанными, переключитесь на `pulse`, чтобы проверить работу движка анимации
