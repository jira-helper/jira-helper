# Дни в колонке (Days in Column)

| | |
|---|---|
| Где настраивается | «Настройки доски» → «Дополнительные элементы карточек» → «Бейдж «Дней в колонке»» |
| Где видно | Доска (детальный вид) |
| Настройки действуют | Для всей команды |

## Цель

Показать, сколько дней задача находится в текущей колонке, с цветовой индикацией порогов «внимания» для быстрого выявления застоявшихся задач.

<DaysInColumnMockup />

Переключите **«Show days badge»** чтобы сменить компактные точки-индикаторы на читаемые бейджи с количеством дней. Зелёный = свежая (≤3 дн), оранжевый = застоялась (≤7 дн), красный = старая (>7 дн).

## Как настроить

### Где найти настройки

1. Откройте доску, затем **«Jira Helper»** → «Настройки доски».
2. Перейдите на вкладку **«Дополнительные элементы карточек»**.

### Как настроить

- **«Включить дополнительные элементы»**: включите главный переключатель **«Включить дополнительные элементы»**.
- **«Выбрать колонки»**: отметьте колонки, в которых должен отображаться бейдж, в секции **«Выбор колонок»**.
- **«Включить бейдж»**: включите **«Показывать бейдж с днями в колонке»** в секции **«Бейдж «Дней в колонке»»**.
- **«Выбрать режим порогов»**:
  - **«Глобальные пороги»** — одинаковые значения для всех колонок.
  - **«Отдельные правила для каждой колонки»** — разные пороги для разных этапов (например, тестирование — 3 дня, разработка — 10 дней).
- **«Задать порог «Warning»»** (жёлтый): бейдж станет жёлтым после этого количества дней. Оставьте поле пустым, если подсветка не нужна.
- **«Задать порог «Danger»»** (красный): бейдж станет красным после этого количества дней. Оставьте поле пустым, если подсветка не нужна.

Стандартный счётчик дней Jira (`.ghx-days`) автоматически скрывается при включении фичи.

Нажмите **«Сохранить»**, чтобы сохранить настройки для всей команды на этой доске.

## Как использовать

- Бейдж отображается в конце карточки в выбранных колонках.
- **Формат текста**: `<1 дн. в колонке`, `1 дн. в колонке`, `X дн. в колонке`.
- **Цвета**: синий (норма), жёлтый (warning), красный (danger) — в зависимости от настроенных порогов.
- При использовании порогов по колонкам для каждой колонки действуют свои значения.
- В бэклоге бейдж не отображается.

## Сценарии использования

1. **Выявление застоявшихся задач** — красный бейдж сразу привлекает внимание к задачам, которые слишком долго находятся в одной колонке.
2. **Контроль SLA по этапам** — разные пороги для разных колонок (например, Code Review — 1 день, тестирование — 3 дня).
3. **Оценка потока** — жёлтые бейджи сигнализируют о задачах, приближающихся к критическому времени пребывания.
