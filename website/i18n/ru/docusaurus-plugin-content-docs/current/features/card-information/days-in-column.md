# Дней в колонке

## Обзор

Бейдж «Дней в колонке» показывает, сколько дней задача находится в текущей колонке доски. Цветной бейдж (синий, жёлтый или красный) позволяет легко заметить стареющие задачи и «застрявшую» работу с первого взгляда.

**Зачем:** Без этой функции вам нужно наводить курсор на каждую карточку или переходить в задачу, чтобы узнать, как долго она находится в колонке. Бейдж отображает данные о старении непосредственно на карточке с настраиваемыми порогами предупреждений для выделения задач, требующих внимания.

**Где:** Настройки доски → Jira Helper → Дополнительные элементы карточек → раздел Дней в колонке. Бейдж отображается на карточках только в выбранных колонках.

**Как это работает:**
- Вычисляет время с последнего перехода между колонками (на основе истории изменений Jira)
- «День» определяется как 24-часовой период с момента последнего перехода, используя **часовой пояс экземпляра Jira** (не локальный часовой пояс пользователя)
- Цветовые пороги могут быть глобальными (одинаковыми для всех колонок) или для каждой колонки отдельно (разные правила для разных этапов рабочего процесса)
- Формат: `<1 дня` / `1 день` / `X дней` для 0, 1 и 2+ дней соответственно
- Цветовая схема: **синий** (ниже предупреждения), **жёлтый** (достигнут порог предупреждения), **красный** (достигнут порог опасности)
- Встроенный счётчик дней Jira на карточках автоматически скрывается при включении этой функции
- Не отображается в представлении бэклога (в бэклоге нет колонок)
- Счётчики обновляются при каждой загрузке доски и при перемещении карточек в другую колонку

**Необходимые условия:**
- Для включения и настройки функции требуются права **администратора доски**
- Функция должна быть включена в разделе **Дополнительные элементы карточек** на панели настроек Jira Helper

## Пользовательские сценарии

### Включение и выбор колонок для отображения бейджей

**Цель:** Включить бейдж «Дней в колонке» и выбрать, в каких колонках он отображается.

1. Откройте доску и нажмите **Jira Helper** на панели инструментов доски.
2. Выберите вкладку **Дополнительные элементы карточек**.
3. Установите флажок **Включить дополнительные элементы карточек**.
4. В разделе **Настройки колонок** выберите колонки, в которых должны отображаться бейджи.
5. Прокрутите до подраздела **Бейдж дней в колонке**.
6. Установите флажок **Показывать бейдж дней в колонке**.
7. Бейджи теперь отображаются на всех карточках в выбранных колонках.

### Установка глобальных порогов предупреждения/опасности (жёлтый/красный дни)

**Цель:** Применить одинаковые пороги старения ко всем выбранным колонкам.

1. В настройках Дней в колонке убедитесь, что **Использовать отдельные правила для каждой колонки** не отмечено.
2. **Порог предупреждения:** Введите количество дней, после которого бейдж становится жёлтым. Оставьте пустым, чтобы отключить жёлтую подсветку.
3. **Порог опасности:** Введите количество дней, после которого бейдж становится красным. Оставьте пустым, чтобы отключить красную подсветку.
4. Если порог опасности меньше или равен порогу предупреждения, появляется иконка предупреждения — но настройки всё равно можно сохранить.
5. Бейджи во всех отслеживаемых колонках теперь следуют этим глобальным порогам.

### Настройка порогов для каждой колонки

**Цель:** Установить разные правила старения для разных этапов рабочего процесса (например, тестирование желтеет через 2 дня, разработка через 5 дней).

1. В настройках Дней в колонке отметьте **Использовать отдельные правила для каждой колонки**.
2. Для каждой выбранной колонки появляется строка.
3. Для каждой строки колонки установите:
   - **Порог предупреждения:** дни для жёлтого бейджа
   - **Порог опасности:** дни для красного бейджа
4. Если колонка была удалена с доски, её строка показывает предупреждение и появляется кнопка **Удалить** для очистки устаревших настроек.
5. Бейджи теперь используют пороги для каждой колонки. Колонки без явных порогов всегда отображаются синим.

### Представление доски: цветной бейдж на карточке

**Цель:** Считывать статус старения с первого взгляда на карточках доски.

1. Посмотрите на карточки в выбранных колонках. Бейдж отображается рядом с концом каждой карточки.
2. Прочитайте текст бейджа:
   - `<1 дня` — задача только что попала в эту колонку
   - `1 день` — один полный день в колонке
   - `3 дня` — три полных дня в колонке
3. Интерпретируйте цвет бейджа:
   - **Синий:** пороги не заданы, или количество дней ниже порога предупреждения
   - **Жёлтый:** количество дней достигло или превысило порог предупреждения
   - **Красный:** количество дней достигло или превысило порог опасности
4. Нажмите на любую карточку с жёлтым или красным бейджем для исследования и принятия мер (продвинуть, разблокировать или переназначить).

## Устранение неполадок

### Бейдж не отображается на карточках

1. Убедитесь, что **Включить дополнительные элементы карточек** отмечено в настройках Jira Helper.
2. Проверьте, что **Показывать бейдж дней в колонке** отмечено в подразделе Дней в колонке.
3. Проверьте, что колонка, в которой находится карточка, **выбрана** в разделе Настройки колонок. Бейджи отображаются только в явно выбранных колонках.
4. Убедитесь, что вы находитесь в детальном представлении доски — бейджи не отображаются в Бэклоге.
5. Для недавно перемещённых карточек обновите страницу доски, чтобы увидеть обновлённые счётчики.

### Количество дней кажется устаревшим или некорректным

1. Счётчик вычисляется на основе **истории изменений** Jira — а не с момента первого появления карточки на доске. Если задача была перемещена из колонки и обратно, счётчик сбрасывается.
2. Функция использует **часовой пояс экземпляра Jira** для расчёта дней. Граница дня основана на серверном времени, которое может отличаться от вашего локального часового пояса.
3. Счётчики обновляются при загрузке доски и после перемещения карточек. Если карточка не перемещалась недавно, обновите доску.

### Встроенный счётчик дней Jira всё ещё виден

1. Когда функция Дней в колонке включена, jira-helper автоматически скрывает встроенный счётчик дней на карточках Jira. Если видны оба, попробуйте обновить доску.
2. В редких случаях обновления DOM Jira могут вызвать временное повторное появление встроенного счётчика — повторное обновление обычно решает проблему.

### Иконка предупреждения в настройках (опасность ≤ предупреждение)

1. Иконка предупреждения информационная — она предупреждает, что порог опасности не больше порога предупреждения.
2. Это означает, что жёлтое и красное состояния сработают при одинаковом количестве дней, делая жёлтый фактически невидимым.
3. Установите порог опасности выше порога предупреждения, чтобы получить различимые жёлтое и красное состояния.
