# Обзор функций

Jira Helper — это браузерное расширение, привносящее профессиональные возможности Kanban в Jira: WIP-лимиты на колонках, swimlane и исполнителях; полноцветную подсветку карточек; временные шкалы Ганта; и аналитические наложения. Оно устраняет разрыв между гибким учётом задач Jira и структурированными элементами управления потоком, необходимыми командам для управления потоком, выявления узких мест и предотвращения перегрузки.

Все функции опциональны и настраиваются для каждой доски отдельно. Настройки могут быть общими для команды (через свойства доски Jira) или личными (локальное хранилище), в зависимости от функции. См. [Board Properties & Local Storage](/docs/settings) для полной разбивки.

> **Впервые с Jira Helper?** Начните с [руководства по быстрому старту](/docs/getting-started/quick-start).

**Термины Jira, используемые ниже:**
- **JQL** — Jira Query Language; SQL-подобный синтаксис для фильтрации задач (например, `priority = High`)
- **Swimlane** — горизонтальные строки на доске, категоризирующие задачи по исполнителю, эпику или пользовательским критериям

## WIP-лимиты

Контролируйте объём незавершённой работы на нескольких уровнях для предотвращения перегрузки команды и выявления узких мест.

- **[Column Group WIP Limits](/docs/features/wip-limits/column-limits)** — Общий WIP-лимит на несколько колонок (CONWIP: Constant Work In Progress). Используйте, когда этап вашего рабочего процесса охватывает несколько колонок и вы хотите ограничить общий объём работы на этом этапе.
- **[Swimlane WIP Limits](/docs/features/wip-limits/swimlane-limits)** — Индивидуальные лимиты на каждую дорожку (например, 3 для Expedite, 10 для Normal). Позволяет задать разные ограничения пропускной способности для разных потоков работ на одной доске.
- **[Personal WIP Limits](/docs/features/wip-limits/personal-limits)** — Ограничение количества задач, с которыми каждый участник команды может работать одновременно. Помогает сбалансировать нагрузку и предотвращает узкие места на уровне отдельных сотрудников.
- **[Field Value WIP Limits](/docs/features/wip-limits/field-limits)** — Распределение пропускной способности на основе значений любого поля задачи (например, макс. 5 на компонент, макс. 3 на приоритет). Для кросс-функциональных ограничений пропускной способности.
- **[Cell WIP Limits](/docs/features/wip-limits/cell-limits)** — WIP-лимиты на пересечениях колонок и swimlane для детального контроля. Полезно, когда разные swimlane имеют разные правила потока в пределах одной колонки.

## Визуализация доски

Улучшите читаемость доски с помощью визуальных подсказок, отражающих приоритет, распределение и статус.

- **[Card Colors](/docs/features/board-visualization/card-colors)** — Заполняет весь фон карточки цветом, заданным родными JQL-правилами Card Colors Jira. Делает различия по приоритету и типу сразу заметными на загруженных досках, где тонкую цветную полоску Jira по умолчанию легко не заметить.
- **[Swimlane Histogram](/docs/features/board-visualization/swimlane-histogram)** — Столбчатая диаграмма, показывающая распределение задач по swimlane. Быстро выявляет перегруженные дорожки и неравномерное распределение работы.

## Информация на карточках

Отображайте дополнительные метаданные непосредственно на карточках задач, чтобы сократить количество кликов и улучшить контекст.

- **[Days in Column](/docs/features/card-information/days-in-column)** — Показывает, как долго задача находится в текущей колонке. Помогает выявить застрявшие элементы и устаревающую работу, требующую внимания.
- **[Days to Deadline](/docs/features/card-information/days-to-deadline)** — Обратный отсчёт до срока задачи с цветовой кодировкой (зелёный → жёлтый → красный по мере приближения дедлайна). Сроки всегда на виду без необходимости открывать каждую задачу.
- **[Issue Links Display](/docs/features/card-information/issue-links-display)** — Отображает ключи и заголовки связанных задач прямо на карточках. Избавляет от необходимости открывать задачу только для того, чтобы увидеть, с чем она связана.
- **[Issue Condition Checks](/docs/features/card-information/issue-condition-checks)** — Отображает пользовательские значки на карточках на основе JQL-условий (например, значок предупреждения, когда не назначен рецензент). Используется для автоматических индикаторов соответствия.

## Прогресс и планирование

Отслеживайте завершение работы и визуализируйте временные шкалы за пределами вида доски.

- **[Sub-tasks Progress](/docs/features/sub-tasks-progress)** — Индикаторы прогресса на родительских карточках, показывающие долю завершённых подзадач или дочерних элементов эпика. Мгновенно показывает, насколько родительская задача близка к завершению без углубления в детали.
- **[Gantt Chart](/docs/features/gantt-chart)** — Интерактивная временная шкала на любой странице задачи, показывающая связанные задачи (подзадачи, эпики, связанные задачи) с масштабированием, панорамированием, разбивкой по статусам и настраиваемыми источниками дат. Полезно для планирования релизов и отслеживания зависимостей.

## Аналитика

Накладывайте опорные линии и измерительные инструменты на родные графики Jira для улучшения оценки и анализа SLA.

- **[SLA Line](/docs/features/control-chart/sla-line)** — Добавляет линию процентиля (например, 85-й процентиль) на Control Chart Jira. Визуализируйте, как ваши фактические времена цикла соотносятся с целевыми показателями SLA.
- **[Scale Ruler](/docs/features/control-chart/scale-ruler)** — Накладывает измерительную сетку на Control Chart для анализа оценки и прогнозирования.

## Утилиты

Инструменты повышения продуктивности, не вписывающиеся в вышеуказанные категории рабочих процессов.

- **[Flag on Issue Panel](/docs/features/flag-issue)** — Добавляет визуальные индикаторы флагов на панели задач для быстрого сканирования срочности.
- **[Comment Templates](/docs/features/issue-templates/comment-templates)** — Сохраняйте и вставляйте многоразовые шаблоны комментариев одним кликом. Сократите повторяющийся ввод для статус-апдейтов, подтверждений QA или сообщений Go/No-Go.
- **[Data Blurring](/docs/features/data-blurring)** — Размывает конфиденциальный текст (ключи задач, заголовки, имена) на доске. Полезно для скриншотов во время презентаций или демонстрации экрана без раскрытия данных клиентов.
- **[Local Settings](/docs/features/local-settings)** — Язык и личные настройки, хранящиеся только в вашем браузере.
