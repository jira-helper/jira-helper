# FAQ

## Общие вопросы

### Что такое Jira Helper и бесплатен ли он?

Jira Helper — это бесплатное браузерное расширение с открытым исходным кодом (лицензия ISC), добавляющее профессиональные функции Kanban в Jira: WIP-лимиты, раскраску карточек, диаграммы Ганта, аналитику и многое другое. Работает как с Jira Cloud, так и с Jira Server/Data Center. Никаких платных тарифов, подписок, премиум-функций — бесплатно для использования, изменения и распространения.

### Какие браузеры поддерживаются?

Chrome 88+, Firefox 58+ и Edge 88+ (на базе Chromium). Расширение использует стандартные API WebExtensions для широкой совместимости.

### Отправляет ли расширение мои данные куда-либо?

**Нет.** Все данные остаются в вашем браузере. Расширение взаимодействует только с вашим экземпляром Jira через его существующий REST API. Никакие данные не передаются третьим лицам, никакой аналитики, телеметрии, внешних серверов. См. [Установка](/docs/getting-started/installation) для примечания о приватности.

### Работает ли с Jira Cloud и Jira Server?

Да. Jira Helper работает с Jira Cloud, Jira Server и Jira Data Center.

### Работает ли на досках Jira Service Management (JSM)?

Нет. Jira Helper предназначен для Scrum- и Kanban-досок разработки. Доски JSM используют другую структуру интерфейса и не поддерживаются.

## Установка и удаление

### Как установить Jira Helper?

См. [руководство по установке](/docs/getting-started/installation). Это занимает меньше минуты: откройте Chrome Web Store или страницу дополнений Firefox, нажмите установить и откройте любую Jira-доску.

### Как удалить?

- **Chrome/Edge:** Перейдите в `chrome://extensions`, найдите Jira Helper, нажмите **Удалить**
- **Firefox:** Перейдите в `about:addons`, найдите Jira Helper, нажмите **Удалить**

Настройки, хранящиеся в **свойствах доски**, не удаляются при удалении расширения — они находятся на сервере Jira. Настройки в **локальном хранилище** удаляются вместе с расширением.

### Можно ли установить на управляемый/корпоративный Chrome?

Да, но вам может потребоваться, чтобы IT-администратор принудительно установил или добавил расширение в белый список. ID расширения: `egmbomekcmpieccamghfgjgnlllgbgdl`. См. [Устранение неполадок установки](/docs/getting-started/installation#troubleshooting).

## Права доступа

### Нужны ли особые права Jira для использования функций?

Зависит от функции:

- **Функции только для просмотра** (Swimlane Histogram, Flag on Issue Panel, Scale Ruler, Data Blurring) — особые права не требуются; любой пользователь с доступом к доске может их использовать
- **Настраиваемые общекомандные функции** (Column WIP Limits, Card Colors, Days in Column, Sub-tasks Progress и т.д.) — требуют прав **администратора доски** для сохранения настроек. Пользователи с правами просмотра могут видеть настройки, но не могут их изменять
- **Личные функции** (Gantt Chart, Comment Templates, Language) — особые права не требуются; хранятся в вашем браузере

### Почему я не могу сохранить настройки WIP-лимитов?

Скорее всего, у вас нет прав **администратора доски**. Попросите администратора Jira предоставить вам доступ администратора доски. Подробнее см. [Board Properties & Local Storage](/docs/settings).

## Функции

### Можно ли использовать WIP-лимиты и диаграмму Ганта вместе?

Да. Все функции независимы и могут свободно комбинироваться.

### Нужно ли настраивать каждую функцию?

Нет. Каждая функция опциональна. Некоторые работают «из коробки» без настройки (Swimlane Histogram, Flag on Issue Panel, Scale Ruler), другие требуют настройки в параметрах доски.

### WIP-лимиты не отображаются на доске

1. Убедитесь, что вы нажали **Save** в панели настроек после настройки
2. **Обновите** страницу доски (F5)
3. Проверьте, что у вас есть права **администратора доски**
4. Проверьте, что функция **включена** (у некоторых есть переключатель вкл/выкл)
5. Убедитесь, что вы на **виде доски**, а не в бэклоге

### Счётчик группы колонок не совпадает с тем, что я вижу

1. Проверьте **фильтр swimlane** группы — он может исключать некоторые swimlane, которые вы видите
2. Проверьте **фильтр типов задач** группы — могут учитываться только определённые типы задач
3. Счётчик учитывает задачи во всех колонках группы, а не только в первой колонке
4. Помните, что фильтры по умолчанию исключают подзадачи в некоторых сценариях

### Диаграмма Ганта показывает неверные даты или не все задачи

1. Проверьте **сопоставление дат** (нажмите собственную шестерёнку ⚙ диаграммы Ганта в заголовке Ганта, чтобы открыть её настройки): для каждой задачи побеждает первый подходящий источник
2. Проверьте **настройки включения задач**: подзадачи, эпики, связанные задачи — все могут переключаться
3. Проверьте **фильтры исключения** — одно совпадение с фильтром удаляет задачу из диаграммы
4. Убедитесь, что источники дат на основе статусов точно соответствуют фактическим названиям статусов вашего рабочего процесса

### Цвета карточек не применяются / карточка не закрашивается целиком

1. Сначала должны быть настроены родные JQL-правила **Card Colors** Jira (настройки доски → Card Colors)
2. Проверьте, что флажок **Fill whole card** включён в настройках Card Colors Jira Helper
3. Отмеченные/заблокированные карточки сохраняют свой исходный стиль — это сделано намеренно
4. Карточки без подходящего JQL-правила остаются белыми

### Функция не работает после обновления интерфейса Jira

Jira периодически обновляет интерфейс, что может временно нарушить работу функций расширения. Команда отслеживает изменения Jira и выпускает исправления. Сообщайте о проблемах через GitHub (см. [Как сообщить об ошибке](#как-сообщить-об-ошибке) ниже).

## Сброс и очистка данных

### Как сбросить все настройки Jira Helper?

Кнопки сброса в один клик нет. Настройки необходимо очищать вручную в зависимости от места хранения:

- **Настройки в свойствах доски:** Откройте панель Jira Helper на каждой доске и удалите конфигурацию вручную (удалите группы, очистите лимиты и т.д.), затем сохраните
- **Настройки в локальном хранилище:** Очистите данные расширения для Jira Helper в браузере или удалите и переустановите расширение

### Настройки потеряны после очистки данных браузера

Настройки, хранящиеся в **свойствах доски**, сохраняются при очистке данных браузера, поскольку они находятся на сервере Jira. Настройки, хранящиеся в **локальном хранилище**, теряются при очистке данных браузера — вам потребуется перенастроить эти функции. См. [Board Properties & Local Storage](/docs/settings), чтобы узнать, какие функции используют какое хранилище.

## Известные ограничения

- **Мобильная Jira** не поддерживается. Jira Helper предназначен только для десктопных браузеров.
- **Доски Service Management (JSM)** не поддерживаются (см. [Работает ли на досках JSM?](#работает-ли-на-досках-jira-service-management-jsm)).
- **Personal WIP Limits** учитывают только текущую видимую доску — они не агрегируют данные по нескольким доскам.
- **Производительность диаграммы Ганта** может снижаться при более чем ~200 связанных задачах на одной странице.
- **Data Blurring** — только визуальное; не предотвращает копирование или извлечение исходного текста.

Сообщайте об ошибках и предложениях на [GitHub Issues](https://github.com/anomalyco/jira-helper/issues).

Пожалуйста, укажите:
- Браузер и версию (Chrome/Firefox/Edge)
- Тип Jira (Cloud или Server/Data Center)
- Шаги для воспроизведения
- Ожидаемое и фактическое поведение
- Скриншоты, если возможно
