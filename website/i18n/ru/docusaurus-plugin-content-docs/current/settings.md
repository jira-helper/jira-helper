# Свойства доски и локальное хранилище

Jira Helper хранит настройки в двух местах в зависимости от функции. Понимание разницы важно: то, что вы настраиваете, может влиять только на вас или на всю вашу команду.

## Свойства доски (общие для команды)

**Свойства доски** (Board Properties) — это встроенное хранилище Jira типа «ключ-значение», привязанное к каждой доске. Когда функция хранит свою конфигурацию в свойствах доски, **значения настроек** являются общими для всех участников доски — вся ваша команда видит одни и те же WIP-лимиты, цвета карточек и другую конфигурацию при просмотре доски. Каждый пользователь по-прежнему видит свой собственный интерфейс настроек; общими являются только сами данные.

<!-- SCREENSHOT: Jira board settings dialog showing the "Column group WIP limits" section with a callout "Settings stored in board properties — visible to all board members" -->

> **Требуются права:** Вы должны быть **администратором доски**, чтобы сохранять настройки, хранящиеся в свойствах доски. Пользователи с правами просмотра могут видеть настройки, но не могут их изменять. Если вы не администратор доски, попросите администратора Jira предоставить вам доступ.

## Локальное хранилище (личное)

**Локальное хранилище** (Local Storage) — это хранилище вашего браузера для расширения. Настройки здесь применяются только к **вашему браузеру** — никто из вашей команды их не видит, и они не синхронизируются между устройствами.

<!-- SCREENSHOT: The Gantt Chart settings modal with a callout "Settings stored locally — your personal configuration only" -->

> **Специальные права не требуются.** Любой пользователь может настраивать функции локального хранилища независимо от своей роли в Jira.

### Функции, использующие локальное хранилище

- **[Gantt Chart](/docs/features/gantt-chart)** — сопоставление дат, цвета полос, правила исключения
- **[Comment Templates](/docs/features/issue-templates/comment-templates)**
- **[Data Blurring](/docs/features/data-blurring)** — состояние переключателя размытия
- **[Local Settings](/docs/features/local-settings)** — языковые настройки

### Функции без настроек

Следующие функции не имеют настраиваемых параметров и работают «из коробки» без настройки:

- **Swimlane Histogram** — автоматически отображает столбчатую диаграмму на доске
- **Flag on Issue Panel** — всегда активен на панелях задач
- **Scale Ruler** — всегда активен на Control Chart

## Доступ к настройкам

1. Откройте любую Jira-доску
2. Нажмите кнопку **Jira Helper** на панели инструментов доски
3. Каждая функция, поддерживающая настройку, имеет свою вкладку в панели настроек: **Columns**, **Swimlanes**, **Card Colors** и т.д.

Дополнительные пути доступа для отдельных функций:
- **Gantt Chart** — имеет собственную отдельную кнопку настроек ⚙ в заголовке диаграммы Ганта. Нажмите эту шестерёнку, чтобы открыть настройки Ганта (сопоставление дат, цвета полос, правила исключения).
- **Comment Templates** — настраиваются через вкладку **Comment Templates** в панели настроек Jira Helper, затем вставляются из выпадающего списка в редакторе комментариев Jira.

## Устранение неполадок

### Общие настройки не применяются для других участников команды

1. Убедитесь, что у участников вашей команды расширение **установлено и включено**
2. Попросите их **обновить** Jira-доску (F5) после того, как вы сохраните изменения
3. Убедитесь, что вы успешно сохранили настройки (откройте панель настроек повторно — ваша конфигурация должна быть на месте)
4. Проверьте, не требует ли функция дополнительной настройки для каждого пользователя (некоторые функции имеют как общие, так и личные переключатели)

### Изменения откатываются после сохранения

Убедитесь, что у вас есть права **администратора доски**. Запись в свойства доски завершается без ошибок, если у вас нет доступа на редактирование. Проверьте панель администрирования вашей Jira-доски, чтобы подтвердить вашу роль.

### Сохранение не срабатывает без сообщения об ошибке

Свойства доски имеют ограничение по размеру, установленное Jira (обычно 32 КБ на свойство). Если ваша конфигурация велика (много групп WIP-лимитов, сложные правила цветов карточек или большие списки проверок условий), сохранение может не сработать без ошибок, поскольку сериализованные настройки превышают этот лимит.

Решение:
- Уменьшите количество групп, правил или условий
- Сократите имена и JQL-запросы
- Разделите сложные конфигурации между функциями, где это возможно

### Настройки потеряны после очистки данных браузера

Настройки, хранящиеся в **свойствах доски**, сохраняются при очистке данных браузера, поскольку они находятся на сервере Jira. Настройки, хранящиеся в **локальном хранилище**, теряются при очистке данных браузера — вам потребуется перенастроить эти функции. Подробнее см. в [FAQ](/docs/advanced/faq#settings-lost-after-clearing-browser-data).
