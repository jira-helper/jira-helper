# Request: Gantt-диаграмма по сабтаскам задачи

**Дата**: 2026-04-11
**Slug**: `gantt-chart`
**Тип**: UI
**Has UI**: yes
**Scope**: large

## Описание запроса

Хочу отображать гант-диаграмму по всем сабтаскам задачи.

Сабтасками могут быть разные задачи — и прямые сабтаски, и линки, и эпик линки.

Пользователь должен мочь настроить:
- Что считать за старт и конец работ
- Какие задачи исключать из ганта

Выбранные настройки должны сохраняться для пользователя.

Гант-диаграмму показываем на новом экране — экран просмотра задачи (issue view). Сейчас там нет фичей от расширения. Нужно придумать куда вставить настройки и как отображать.

По гант-диаграмме — пока надо уметь просто отображать колбаски, но в будущем возможно нужно будет научиться и менять что-то, но точно не сейчас.

## Уточнения

### Раунд 1

**Старт и конец работ:**
- В основном поля с датами, но было бы круто поддерживать переходы в статусы.
- Для задач без конечного статуса — непонятно как строить end.
- Возможно стоит иметь режим с разбивкой по статусам внутри колбаски.

**Исключение задач:**
- Использовать существующий компонент `IssueSelectorByAttributes` (простой выбор по полям + JQL).

**Размещение на странице:**
- Под основным блоком задачи (полная ширина).

**Информация на колбаске:**
- Настраиваемое. По умолчанию — ключ задачи.
- Пользователь может выбрать любое поле.
- Возможность выбрать поля для отображения при наведении (подробности).

**Настройки:**
- Модалка настроек (открывается по кнопке ⚙️).

### Раунд 2

**Разбивка по статусам:**
- Toggle: по умолчанию простые колбаски, можно включить разбивку по статусам.

**Задачи без дат:**
- Есть старт, нет конца → колбаска до «сегодня» + иконка warning.
- Нет ни старта, ни конца → отдельная секция под Гантом «N задач не отображены» (сворачиваемая, с причинами).

**Версия Jira:**
- Только classic Jira (Server/DC).

**Библиотека:**
- Предложено: чистый CSS/HTML + React (без внешних библиотек).
- Полный контроль, нет раздувания бандла, легко добавить статусные секции и будущие фичи.

### Раунд 3

**Zoom/Pan:**
- Нужен zoom in/out, выбор интервалов (hours, days, weeks, months).
- Pan как на доске Miro (drag + scrollbars).
- Wheel zoom + кнопки +/- + dropdown интервала.

**Библиотека (пересмотр):**
- Бандл не важен (расширение браузера, не из сети).
- Решение: React + SVG + d3-scale/d3-zoom (гибридный подход).

**Производительность:**
- Все задачи отображаем as-is, без пагинации. Не должно лагать.

**Accessibility:**
- Не интересует. Только browser Find (Ctrl+F) по тексту колбасок.

**Default mapping:**
- Нет default — пользователь должен настроить сам. До настройки — сообщение «Настройте параметры».

**Scope ключа localStorage:**
- Глобальный ключ, настройки per project.

**Кнопка gear:**
- Не фиксировать расположение, PageObject предоставит метод `insertSettings`.

### Правки к requirements (от пользователя)

**FR-4 (Разбивка по статусам):**
- Цвета секций — цвета категорий статусов, аналогично прогресс-барам (`jiraColorScheme`).

**FR-9 (Настройки — каскадная модель):**
- По умолчанию настройки глобальные, но пользователь может переопределить per project или per project+issueType.
- Каскад: `Project+IssueType` > `Project` > `Global`.
- В модалке — scope selector + «Copy from...» для копирования настроек между scope.
