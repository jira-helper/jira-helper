# Тест-план: WIP Limits в Jira Helper

## 1. Person Limits (Персональные лимиты)

### 1.1 Настройки (Settings Page)

#### Добавление лимита
- [ ] **SC1**: Добавить новый лимит для пользователя (ввести имя, установить лимит 5, нажать "Add limit" → пользователь появился в списке)
- [ ] **SC2**: Добавить лимит только для определённых колонок (выбрать только "To Do, In Progress" → лимит применяется только к этим колонкам)

#### Редактирование лимита
- [ ] **SC3**: Редактировать существующий лимит (нажать "Edit" → имя и лимит отображаются в полях, кнопка показывает "Edit limit")
- [ ] **SC4**: Обновить значение лимита (изменить с 5 на 10, нажать "Edit limit" → лимит обновлён)

#### Удаление лимита
- [ ] **SC5**: Удалить лимит (нажать "Delete" → пользователь удалён из списка)

#### Массовые операции
- [ ] **SC6**: Применить колонки к нескольким лимитам (выбрать несколько пользователей, "Apply columns" → колонки применены ко всем)

#### Swimlanes
- [ ] **SC7**: Добавить лимит только для определённых swimlanes (выбрать "Frontend" → лимит применяется только к этому swimlane)
- [ ] **SC8**: Применить swimlanes к нескольким лимитам массово

#### Типы задач
- [ ] **SC9**: Добавить лимит только для определённых типов задач (снять "Count all issue types", выбрать "Task, Bug")
- [ ] **SC10**: При редактировании сохраняются выбранные типы задач

#### Валидация
- [ ] **SC11**: Нельзя добавить лимит без имени пользователя (показывается ошибка)
- [ ] **SC12**: Нельзя добавить лимит с нулевым значением (показывается ошибка)

#### Отмена редактирования
- [ ] **SC13**: Отмена редактирования возвращает к режиму добавления (кнопка "Add limit", значения не изменены)

#### Пустое состояние
- [ ] **SC14**: При отсутствии лимитов показывается пустая таблица и предупреждение об аватарах

#### Несколько лимитов на пользователя
- [ ] **SC15**: Добавить несколько лимитов для одного пользователя с разными колонками

### 1.2 Доска (Board Page)

#### Отображение счётчика
- [ ] Показать счётчик карточек для пользователя с лимитом (4 из 5 → "4 / 5", зелёный)
- [ ] Подсветить пользователя при превышении лимита (4 из 3 → "4 / 3", красный, карточки подсвечены)

#### Фильтры колонок и swimlanes
- [ ] Применить лимит только к определённым колонкам (считаются только карточки в col2)
- [ ] Применить лимит только к определённым swimlanes (считаются только карточки в sw1)

#### Фильтр по типам задач
- [ ] Применить лимит только к Bug и Task (Story не считается)

#### Фильтрация по клику
- [ ] Клик по аватару показывает только карточки этого пользователя в лимитируемых колонках
- [ ] Клик по второму лимиту того же пользователя показывает только карточки для этого лимита

## 2. WIP Limit on Cells (Лимиты на ячейках)

### 2.1 Настройки (Settings Page)

#### Открытие/Закрытие попапа
- [ ] **SC1**: Открыть попап настроек (нажать "Edit Wip limits by cells" → видна форма, dropdown-ы, таблица ranges)
- [ ] **SC2**: Сохранить и закрыть попап (нажать "Save" → попап закрыт, изменения сохранены в Jira property)
- [ ] **SC3**: Отменить без сохранения (нажать "Cancel" → попап закрыт, изменения НЕ сохранены)
- [ ] **SC26**: Закрыть по кнопке X без сохранения

#### Добавление range
- [ ] **SC4**: Добавить новый range с ячейкой (ввести имя, выбрать swimlane/column, добавить → range в таблице)
- [ ] **SC5**: Нельзя добавить range без имени (alert "Enter range name")
- [ ] **SC6**: Нельзя добавить range с дублирующимся именем (alert "Enter unique range name")

#### Добавление ячейки в существующий range
- [ ] **SC7**: Кнопка меняется на "Add cell" при вводе существующего имени range
- [ ] **SC8**: Кнопка показывает "Add range" для нового имени
- [ ] **SC9**: Добавить ячейку в существующий range
- [ ] **SC10**: Нельзя добавить дублирующуюся ячейку в range

#### Валидация
- [ ] **SC11**: Нельзя добавить range без выбора swimlane (alert)
- [ ] **SC12**: Нельзя добавить range без выбора column (alert)

#### Редактирование range
- [ ] **SC13**: Редактировать имя range inline
- [ ] **SC14**: Редактировать WIP лимит inline
- [ ] **SC15**: Переключить checkbox "Disable"
- [ ] **SC16**: Выбрать range для редактирования через иконку edit (имя заполняется в input)

#### Удаление
- [ ] **SC17**: Удалить range целиком
- [ ] **SC18**: Удалить отдельную ячейку из range

#### Очистка всех настроек
- [ ] **SC19**: Нажать "Clear and save all data" → все ranges удалены, property очищено

#### Persistence
- [ ] **SC20**: Сохранение в Jira board property "WIP_LIMITS_CELLS"
- [ ] **SC21**: Загрузка настроек при открытии попапа

#### Обратная совместимость
- [ ] **SC22**: Загрузка настроек с legacy полем "swimline" вместо "swimlane"

#### Show Badge
- [ ] **SC23**: Добавить ячейку с галочкой "show indicator" (иконка badge отображается)
- [ ] **SC24**: Добавить ячейку без галочки "show indicator" (иконка badge НЕ отображается)

#### Пустое состояние
- [ ] **SC25**: Пустая таблица при отсутствии настроек (можно добавить range)

### 2.2 Доска (Board Page)

#### Отображение badge
- [ ] **SC1**: Badge показывает счёт и лимит на ячейке с showBadge=true (другие ячейки без badge)
- [ ] **SC2**: Badge считает карточки по всем ячейкам в range суммарно

#### Цветовые индикаторы badge
- [ ] **SC3**: Зелёный badge при выполнении лимита (#1b855c)
- [ ] **SC4**: Жёлтый badge при достижении лимита (#ffd700)
- [ ] **SC5**: Красный badge при превышении лимита (#ff5630)

#### Фон ячеек
- [ ] **SC6**: Красный фон на ячейках при превышении лимита (class "WipLimit_NotRespected")
- [ ] **SC7**: Без красного фона при выполнении лимита (class "WipLimit_Respected")

#### Пунктирные границы
- [ ] **SC8**: Одиночная ячейка получает границы со всех 4 сторон
- [ ] **SC9**: Соседние ячейки в одной строке — общие внутренние границы
- [ ] **SC10**: Соседние ячейки в одном столбце — общие внутренние границы
- [ ] **SC11**: L-образный range — правильные границы на внешних краях

#### Отключённый range
- [ ] **SC12**: Disabled range показывает диагональную штриховку (class "WipLimitCells_disable")
- [ ] **SC13**: Disabled range показывает границы, но без индикаторов лимита

#### Фильтр по типам задач
- [ ] **SC14**: Считаются только указанные типы задач (Bug, Task считаются, Story нет)
- [ ] **SC15**: Все типы считаются при отсутствии фильтра

#### Несколько ranges
- [ ] **SC16**: Несколько ranges отображаются независимо (разные цвета для разных ranges)

#### Динамическое обновление
- [ ] **SC17**: Badge обновляется при перемещении карточки в ячейку
- [ ] **SC18**: Badge обновляется при перемещении карточки из ячейки

#### Отсутствующие ячейки
- [ ] **SC19**: Несуществующие ячейки пропускаются без ошибок

#### Без настроек
- [ ] **SC20**: Доска отображается нормально без WIP limit настроек (без badges и границ)
