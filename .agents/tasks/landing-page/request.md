# Request: Landing page and documentation site for jira-helper

**Дата**: 2026-05-09
**Slug**: `landing-page`
**Тип**: large
**Has UI**: yes
**Scope**: large

## Описание запроса

Сделать анализ на предмет того, как сделать крутой лендинг для проекта jira-helper.
Что там должно быть?

## Уточнения

### Раунд 1 — цель лендинга

**Вопрос**: Какую основную цель должен решать лендинг?

**Ответ**: Гибрид: и лендинг с фичами, и полноценная документация с описанием каждой фичи, примерами настройки, FAQ.

### Раунд 2 — платформа

**Вопрос**: На какой платформе делать сайт?

**Ответ**: GitHub Pages + static site gen. Просто, бесплатно, рядом с репозиторием.

### Раунд 3 — локализация

**Вопрос**: Нужна поддержка двух языков (RU/EN) на лендинге?

**Ответ**: Да, RU и EN. Переключатель языков. Контент на двух языках параллельно.

### Раунд 4 — выбор генератора

**Вопрос**: Какой SSG?

**Ответ**: Docusaurus. React-стек (консистентно с проектом), версионирование, swizzling для маркетингового лендинга.

### Раунд 5 — структура

Вопрос про структуру сайта и документации.

**Ответ утверждён**:

```
/                          → Лендинг (hero + фичи + CTA)
/docs/                     → Вся документация
  /getting-started/        → Установка, быстрый старт
  /features/               → По фиче:
    /wip-limits/
    /board-visualization/
    /card-information/
    /sub-tasks-progress/
    /issue-templates/
    /gantt-chart/
    /control-chart/
    /flag-issue/
    /data-blurring/
    /local-settings/
  /settings/               → Как работает система настроек
  /advanced/               → JQL-референс, FAQ
```

Ченджлог и контрибьютинг — не нужны.

### Раунд 6 — структура страницы фичи

**Вопрос**: Секции лендинга?

**Ответ**:
1. Hero — что это + CTA на установку
2. Features Grid — 6-8 карточек ключевых фич с гифками/скриншотами
3. Stats / Social Proof — установки, звёзды
4. CTA Block — установить, смотреть доки, GitHub

Живые React-компоненты на лендинге — не в первой версии, только скриншоты.

### Раунд 7 — структура документации фичи

Каждая фича = Overview + User Jobs.
- **Overview**: что делает, скриншот, зачем нужно
- **User Jobs**: по подзаголовку на каждую юзер-джобу (настройка, board view, advanced)

Группировка фич в подпапках утверждена.
