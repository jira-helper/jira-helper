# Requirements: Jira Comment Templates

**Feature folder**: `.agents/tasks/jira-comment-templates/`
**Связанный request**: [request.md](./request.md)
**Дата**: 2026-04-29
**Статус**: agreed
**Has UI**: yes

## 1. Цель и мотивация

Пользователю нужно перенести отдельное Chrome extension `Jira Comment Templates – TCS` внутрь `jira-helper`, чтобы быстрые шаблоны комментариев, их настройка и связанные действия были частью основного расширения.

Фича должна ускорять типовые ответы в Jira: пользователь открывает форму комментария, видит рядом с редактором набор сохранённых шаблонов, кликом вставляет нужный текст и при необходимости автоматически добавляет watchers к задаче.

Реализация должна сохранить полезное поведение исходного расширения, но быть спроектирована по правилам `jira-helper`: DOM-доступ через PageObject, Jira REST через JiraClient, состояние через Model, UI через React-компоненты.

## 2. Пользователи и контекст

- Роли / контекст использования: пользователи Jira, которые часто оставляют типовые комментарии в задачах и хотят быстро вставлять заранее сохранённые тексты.
- Страницы Jira и точки входа: все Jira-инстансы, где работает `jira-helper`; MVP должен покрыть comment form на issue view и board detail panel. Workflow/transition dialog с комментарием вынесен в отдельную research-задачу.
- Основная точка входа: тулбар `Шаблоны` рядом с формой комментария Jira.
- Настройка: кнопка `Шаблоны` / `Управление шаблонами` в тулбаре открывает модальное окно управления шаблонами.

## 3. Функциональные требования

1. Фича должна обнаруживать формы комментариев Jira и добавлять рядом с редактором комментария тулбар шаблонов.
2. Тулбар должен отображать кнопки для всех сохранённых шаблонов.
3. Каждая кнопка шаблона должна показывать пользовательское название и визуальный цвет шаблона.
4. Клик по кнопке шаблона должен вставлять текст шаблона в текущий редактор комментария в позицию курсора или в конец текста, если позиция курсора недоступна.
5. Вставка должна поддерживать Jira wiki editor, который на проверенной Jira представлен контейнером `.jira-wikifield`, `#comment-wiki-edit`, `textarea#comment` и `rich-editor`.
6. Если у шаблона есть watchers, после вставки текста фича должна попытаться добавить этих пользователей в watchers текущей задачи через Jira REST API. Watchers входят в MVP.
7. Результат добавления watchers должен быть показан пользователю через короткое уведомление в правом верхнем углу: success, partial success или warning/error. Уведомление скрывается через 5 секунд.
8. Пользователь должен иметь возможность открыть модальное окно управления шаблонами из тулбара.
9. В модальном окне пользователь должен иметь возможность добавить новый шаблон.
10. В модальном окне пользователь должен иметь возможность изменить название кнопки, текст шаблона, цвет и список watchers.
11. В модальном окне пользователь должен иметь возможность удалить шаблон.
12. Изменения в модальном окне должны сохраняться локально и обновлять все уже вставленные тулбары без перезагрузки страницы.
13. Пользователь должен иметь возможность экспортировать шаблоны в JSON-файл.
14. Пользователь должен иметь возможность импортировать шаблоны из JSON-файла.
15. При ошибке импорта пользователь должен получить понятное сообщение, а текущие сохранённые шаблоны не должны ломаться.
16. Если сохранённых шаблонов нет, фича должна использовать 2 generic шаблона по умолчанию:
  - `Взял в работу` → `Здравствуйте! Задача взята в работу. Вернусь с обновлением, когда появится результат.`;
    - `Нужно уточнение` → `Здравствуйте! Нужно уточнить:` .
17. Пользователь должен иметь возможность сбросить шаблоны к значениям по умолчанию из настроек `jira-helper` / модалки управления шаблонами. Popup исходного расширения не переносится.
18. Фича не должна дублировать тулбар при повторных DOM-мутациях Jira.
19. Фича должна корректно переживать динамическое открытие, закрытие и повторное появление формы комментария.

## 4. Сценарии (happy path + важные края)

### S1: Вставка шаблона в комментарий

- Given пользователь открыл задачу Jira и активировал форму комментария.
- When `jira-helper` обнаружил форму комментария.
- Then рядом с редактором отображается тулбар `Шаблоны` с кнопками доступных шаблонов.
- When пользователь нажимает кнопку шаблона.
- Then текст шаблона вставляется в редактор комментария.

### S2: Шаблон добавляет watchers

- Given у выбранного шаблона заполнен список watchers.
- When пользователь нажимает кнопку шаблона.
- Then текст шаблона вставляется в комментарий.
- And `jira-helper` отправляет запросы в Jira REST API на добавление watchers к текущей задаче.
- And пользователь видит уведомление о результате в правом верхнем углу, которое скрывается через 5 секунд.

### S2a: Комментарий на board detail panel

- Given пользователь открыл Jira board и выбрал задачу в detail panel.
- When пользователь открывает / видит блок комментария.
- Then тулбар шаблонов отображается рядом с editor в `#addcomment`.
- And вставка шаблона работает в поле `textarea#comment` / Jira rich editor.

### S2b: Комментарий на issue view

- Given пользователь открыл экран задачи.
- When пользователь нажимает `Add comment`.
- Then в inline form `#addcomment` отображается тулбар шаблонов.
- And вставка шаблона работает до нажатия Jira-кнопки `Add`.

### S2c: Комментарий при transition в Trashed

- Статус: вне MVP, research task.
- Нужно отдельно исследовать, откуда надёжно брать issue key внутри workflow/transition dialog.
- До решения research-задачи watchers для transition dialog не считаются обязательным поведением.

### S3: Управление шаблонами

- Given пользователь нажал `Шаблоны` / `Управление шаблонами` в тулбаре.
- When открывается модальное окно.
- Then пользователь может добавлять, редактировать, удалять шаблоны, менять цвет и watchers.
- When пользователь сохраняет изменения.
- Then шаблоны сохраняются локально и тулбар обновляется.

### S4: Импорт и экспорт

- Given пользователь открыл модальное окно управления шаблонами.
- When пользователь нажимает экспорт.
- Then браузер скачивает JSON-файл с текущими шаблонами.
- When пользователь выбирает JSON-файл для импорта.
- Then фича валидирует структуру и заменяет draft-список шаблонов в модалке.
- And окончательное сохранение происходит только после явного `Сохранить`.
- And JSON, экспортированный старым расширением `Jira Comment Templates – TCS`, должен импортироваться как совместимый формат.

### S5: Редактор не найден

- Given форма комментария найдена, но подходящий редактор не найден.
- When пользователь нажимает шаблон.
- Then фича не должна падать.
- And пользователь должен получить понятный warning или действие должно быть залогировано. Точное UX-поведение — TBD.

## 5. Данные и миграции

- Источник истины данных для MVP: `localStorage` в контексте Jira/content script, по аналогии с существующими local settings `jira-helper`.
- Начальный ключ исходного расширения: `jira_cnt_templates`.
- Целевой ключ `jira-helper`: `jira_helper_comment_templates` или похожий namespaced ключ, финально выбрать на этапе design.
- Структура шаблона:
  - `id: string`;
  - `label: string`;
  - `color: string`;
  - `text: string`;
  - `watchers?: string[]`.
- Шаблоны по умолчанию должны быть определены в коде фичи: 2 generic comment templates без CNT-брендинга.
- Автоматическая миграция данных из уже установленного отдельного расширения не требуется.
- Импорт должен поддерживать JSON старого расширения `Jira Comment Templates – TCS`: массив шаблонов с полями `id`, `label`, `color`, `text`, `watchers?`.
- Popup не создаётся, поэтому `chrome.storage.local` не требуется для MVP. Если popup появится в будущем, storage нужно будет пересмотреть и вынести в browser extension storage / message bridge.
- Настройки не должны быть board-specific, если пользователь не подтвердит обратное.

## 6. Нефункциональные требования

- Реализация должна соответствовать архитектуре `jira-helper`: PageObject для DOM, JiraClient для REST API, Model для состояния, React View для UI.
- Фича должна быть идемпотентной при повторных DOM-мутациях Jira.
- UI должен наследовать базовые цвета/контраст Jira и не ломаться при отличающейся теме страницы. Отдельная поддержка dark theme не является самостоятельным scope MVP; это наблюдение пришло из исходного `styles.css`, где была логика dark-theme variables.
- Интерактивные элементы должны быть доступны с клавиатуры: фокус, кнопки, закрытие модалки, понятные labels.
- Ошибки Jira API и импорта JSON не должны ломать форму комментария.
- Тестирование:
  - unit tests для моделей и валидации импортируемого JSON;
  - component tests для модалки управления шаблонами;
  - интеграционный / e2e smoke для появления тулбара и вставки шаблона, если доступна стабильная тестовая Jira-страница.

## 7. Вне scope

- Перенос исходного popup. Управление должно быть доступно через настройки `jira-helper` / UI в Jira.
- Синхронизация шаблонов между браузерами или пользователями.
- Серверное хранение шаблонов в Jira issue / project / board properties.
- Автоматическое определение и автозаполнение логинов watchers.
- Массовое управление watchers вне действия конкретного шаблона.
- Изменение существующей фичи `bug-template`, кроме возможного будущего объединения после отдельного решения.

## 8. Открытые вопросы

- Целевая Jira: все Jira-инстансы, где работает `jira-helper`.
- Branding `CNT` не переносим; фича и дефолтные шаблоны должны быть нейтральными / generic.
- Где именно должен работать тулбар в MVP: board issue detail/comment и issue view comment. Transition dialog `Trashed` вынесен в research-задачу.
- Popup расширения не нужен; UX управления должен быть в Jira / настройках `jira-helper`.
- Нужна кнопка reset к умолчаниям в модалке или настройках `jira-helper`, без отдельного popup.
- Импорт JSON загружает шаблоны в draft модалки; сохранение в `localStorage` происходит только после нажатия `Сохранить`.
- Автоматическая миграция не нужна; нужен импорт JSON, экспортированного старым расширением.
- Watchers — обязательная часть MVP.
- По умолчанию нужны 2 нейтральных generic comment templates, чтобы пользователи понимали формат работы.
- Watchers не валидируются предварительно; логины отправляются как указаны, результат обрабатывается по ответу Jira API.
- Notification по результату добавления watchers показывается в правом верхнем углу и скрывается через 5 секунд.

## 9. Черновик критериев приёмки (для EPIC / BDD)

- На issue view рядом с inline form `#addcomment` появляется тулбар шаблонов.
- На board detail panel рядом с comment form `#addcomment` появляется тулбар шаблонов.
- Клик по шаблону вставляет текст в textarea editor.
- Клик по шаблону вставляет текст в Jira rich editor / wiki editor.
- Тулбар не дублируется после DOM-мутаций.
- Пользователь может добавить, изменить и удалить шаблон в модалке.
- Сохранение шаблонов обновляет тулбар без reload страницы.
- Импорт валидного JSON загружает шаблоны.
- Импорт JSON старого расширения `Jira Comment Templates – TCS` загружает шаблоны в draft модалки.
- Импорт валидного JSON не сохраняет данные до нажатия `Сохранить`.
- Импорт невалидного JSON показывает ошибку и не портит текущие данные.
- Экспорт скачивает JSON текущих шаблонов.
- Шаблон с watchers вызывает Jira REST API и показывает результат в правом верхнем углу на 5 секунд.
- Фича покрыта unit/component tests на критичное поведение.
- Для transition dialog issue key создана отдельная research-задача перед добавлением поддержки.

## 10. UI Wireframe

### Тулбар в форме комментария

```
+-- Jira Comment Form ------------------------------+
| Шаблоны  [Взял в работу] [Нужно уточнение] [Шаблоны] |
|                                                     |
| +-----------------------------------------------+   |
| | Написать комментарий...                       |   |
| +-----------------------------------------------+   |
|                                  [Добавить] [X]     |
+-----------------------------------------------------+
```

**Легенда:**

- `[Взял в работу]` — кнопка пользовательского шаблона.
- `[Шаблоны]` — кнопка открытия модалки управления.

### Модальное окно управления

```
+-- Управление шаблонами ----------------------------+
| Цвет | Название кнопки | Текст шаблона | Watchers | |
|------|-----------------|---------------|----------|-|
| (g)  | Взял в работу   | Здравствуйте! | iv.petrov| X|
| (b)  | Нужно уточнение | Нужно уточнить|          | X|
|                                                     |
| [+ Добавить]                                       |
|                                                     |
| [Сохранить на ПК] [Загрузить с ПК]      [Сохранить] |
+-----------------------------------------------------+
```

**Легенда:**

- `(g)` / `(p)` — выбор цвета шаблона.
- `Watchers` — список логинов через запятую.
- `X` — удаление строки.
