# Requirements: Настройки Column WIP Limits (CONWIP) на board page

**Feature folder**: `.agents/tasks/column-wip-inline/`
**Связанный request**: [request.md](./request.md)
**Дата**: 2026-04-04
**Статус**: agreed

## 1. Цель и мотивация

Сейчас настройки CONWIP (групповых WIP-лимитов по колонкам) доступны только через **Board Settings → Columns → Column group WIP limits**. Чтобы изменить лимит, пользователю нужно покинуть доску, перейти в настройки Jira, найти нужную вкладку и открыть модальное окно. Это избыточно для частой операции.

Фича **переносит UI настроек column-limits** из Board Settings в **новый таб** в панели **Jira Helper** на board page. Это НЕ новая фича и НЕ новая модель данных — это альтернативная точка входа для управления уже существующей функциональностью CONWIP.

**Критерий успеха**: пользователь может создать, изменить и удалить группы WIP-лимитов по колонкам, не покидая доску — через таб в панели Jira Helper.

**Тип**: перенос UI существующей фичи в новое место (изменение UX без изменения поведения).

## 2. Пользователи и контекст

- **Роли**: администратор доски (или пользователь с правом `canEdit` на board properties) — настраивает лимиты через таб. Все пользователи доски с расширением — видят индикаторы и подсветку на board page (поведение runtime не меняется).
- **Страница Jira**: agile board page (Scrum / Kanban).
- **Точка входа**: новый **таб** (вкладка) в панели Jira Helper на board page. Панель открывается кликом по иконке Jira Helper в сайдбаре. Таб регистрируется через `registerSettings` (существующий механизм `BoardSettingsStore`).
- **Изоляция**: настройки привязаны к конкретной доске (board property `WIP_LIMITS_SETTINGS`) — как и текущая фича.

## 3. Функциональные требования

1. **FR-1**: В панели Jira Helper на board page появляется новый таб для настройки column WIP limits.
2. **FR-2**: Таб предоставляет ту же функциональность, что и текущая модалка в Board Settings → Columns:
   - Создание, редактирование и удаление групп колонок.
   - Drag-and-drop колонок между группами и зоной «Without Group».
   - Задание числового лимита (`max`) для группы.
   - Выбор swimlanes для группы (all / конкретные).
   - Выбор типов задач (`includedIssueTypes`) для группы.
   - Выбор кастомного цвета (`customHexColor`) для группы.
3. **FR-3**: Кнопки **Save** и **Cancel** работают идентично текущей модалке: Save сохраняет в board property, Cancel отменяет изменения.
4. **FR-4**: Таб доступен только если у пользователя есть право редактировать доску (`canEdit`).
5. **FR-5**: Данные хранятся в том же board property (`WIP_LIMITS_SETTINGS`), что и сейчас. Новых ключей не добавляется.
6. **FR-6**: Список колонок доски читается из DOM board page (не из DOM settings page).
7. **FR-7**: Runtime-поведение на доске (индикаторы, подсветка exceeded, подсчёт задач) — без изменений.

## 4. Сценарии (happy path + важные края)

### S1: Создание группы через таб на board page

- **Given** пользователь на board page, открыл панель Jira Helper, перешёл на таб column WIP limits.
- **When** создаёт новую группу, перетаскивает в неё колонки, задаёт лимит, нажимает Save.
- **Then** настройки сохраняются в board property, на доске появляются индикаторы CONWIP для этих колонок.

### S2: Редактирование существующей группы

- **Given** на доске уже настроены CONWIP-группы (через Board Settings или ранее через таб).
- **When** пользователь открывает таб, изменяет лимит/swimlanes/issue types/цвет группы, нажимает Save.
- **Then** изменения сохраняются, runtime-поведение на доске обновляется.

### S3: Отмена изменений

- **Given** пользователь открыл таб и начал редактировать.
- **When** нажимает Cancel (или закрывает панель без Save).
- **Then** изменения отбрасываются, board property остаётся прежним.

### S4: Нет прав на редактирование

- **Given** пользователь без права `canEdit` на board.
- **When** открывает панель Jira Helper.
- **Then** таб column WIP limits не показывается (аналогично текущему поведению: кнопка в Board Settings не рендерится при отсутствии `canEdit`).

### S5: Совместимость с Board Settings

- **Given** группы созданы через таб на board page.
- **When** пользователь открывает Board Settings → Columns → Column group WIP limits.
- **Then** те же группы отображаются и редактируются — данные одни и те же (board property).

### S6: Пустое состояние

- **Given** нет настроенных CONWIP-групп.
- **When** пользователь открывает таб.
- **Then** все колонки в зоне «Without Group», форма создания группы доступна.

## 5. Данные и миграции

- **Источник истины данных**: Jira Board Property, ключ `WIP_LIMITS_SETTINGS` (тип `WipLimitsProperty = Record<string, ColumnLimitGroup>`).
- **Миграции**: не требуются — формат данных не меняется, ключ property не меняется.
- **Обратная совместимость**: полная. Данные, созданные через Board Settings, работают в табе и наоборот.
- **Список колонок**: получается из DOM board page (вместо DOM settings page). Нужен PageObject для чтения колонок с board page.
- **Список swimlanes**: получается из Board Edit Data API (как и сейчас) или из DOM board page.

## 6. Нефункциональные требования

- **Доступность**: drag-and-drop колонок должен поддерживать клавиатурную навигацию (как в текущей реализации).
- **Тестирование**:
  - Vitest: если появляется новая SettingsUI Model (или переиспользуется существующая) — unit-тесты на неё.
  - Cypress: открытие таба, создание/редактирование/удаление группы, Save/Cancel, drag-and-drop.
  - Storybook: View-компоненты таба (форма, список групп).

## 7. Вне scope

- Изменение runtime-поведения CONWIP на доске (индикаторы, подсветка, подсчёт задач) — остаётся как есть.
- Удаление старой точки входа (Board Settings → Columns) — старая модалка продолжает работать параллельно.
- Новые параметры лимитов (per-column без группы, новые фильтры) — не в этой итерации.
- Swimlane WIP limits, cell WIP limits, person limits — не затрагиваются.

## 8. Открытые вопросы

Нет — все вопросы закрыты ответами пользователя.

## 9. Черновик критериев приёмки (для EPIC / BDD)

- [ ] В панели Jira Helper на board page есть таб для настройки column WIP limits.
- [ ] Через таб можно создать группу колонок с лимитом (drag-and-drop колонок, ввод max).
- [ ] Через таб можно задать swimlanes, типы задач и цвет для группы.
- [ ] Save сохраняет настройки в board property `WIP_LIMITS_SETTINGS`.
- [ ] Cancel отбрасывает изменения без сохранения.

- [ ] Таб не показывается пользователям без `canEdit`.
- [ ] Runtime-поведение на доске (индикаторы, подсветка exceeded) не изменилось.
