# TASK-191: Fix — при удалении колонки из группы не стирается старая шапка

**Status**: TODO

**Parent**: [EPIC-19](./EPIC-19-column-wip-inline-board-tab.md)

---

## Описание

При Save через таб, если колонка была убрана из группы:
- Шапка перерисовывается (новые индикаторы появляются)
- Но старая шапка (стили/индикаторы от предыдущего состояния) НЕ стирается
- Остаётся визуальный артефакт

При добавлении колонки — всё работает корректно.

## Причина (гипотеза)

`boardRuntimeModel.apply()` вызывается после Save. Вероятно, `apply()` добавляет новые элементы, но не очищает старые DOM-модификации (стили заголовков, бейджи) для колонок, которые были убраны из группы.

## Что сделать

1. Изучить `BoardRuntimeModel.apply()` — как он обновляет DOM шапки
2. Найти, где стили/бейджи добавляются на заголовки колонок
3. Добавить очистку старых стилей/элементов ПЕРЕД добавлением новых (или сделать полный clear + re-render)
4. Протестировать: создать группу → Save → убрать колонку → Save → шапка чистая

## Критерии приёмки

- [ ] При удалении колонки из группы и Save — старые стили/индикаторы шапки очищаются
- [ ] При добавлении колонки — поведение не ломается
- [ ] Тесты проходят: `npm test`
- [ ] Нет ошибок линтера: `npm run lint:eslint -- --fix`
