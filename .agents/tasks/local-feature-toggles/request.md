# Request: Локальное отключение Comment Templates и Gantt

**Дата**: 2026-05-16
**Slug**: local-feature-toggles
**Тип**: UI
**Has UI**: yes
**Scope**: large

## Описание запроса

Нужно сделать возможность локально (в текущем браузере пользователя) отключать функционал:

- Comment Templates
- Gantt

Ожидаемое поведение:

1. При выключении фича не применяется в runtime (не рендерится UI и не работают интеграции фичи).
2. Таб с настройками фичи остаётся доступным, но показывает только один toggle ON/OFF.
3. Toggle расположен внутри соответствующего feature-tab, а не в отдельном Local Settings.
4. Доступность табов должна остаться как сейчас:
   - Comment Templates: board + issue settings
   - Gantt: issue settings
5. Дефолт: обе фичи включены (`enabled = true`).
6. Данные фич не удаляются при OFF (шаблоны, настройки Gantt сохраняются).
7. UI toggle-блока: минималистичный, вверху таба; при OFF показывается короткий hint.

## Уточнения

### Раунд 1

- **Размещение toggle**: внутри каждой фичи (два независимых toggle), без общего блока/центрального settings.
- **Runtime-поведение**: OFF должен отключать фичу сразу в текущей сессии.
- **Данные**: при OFF данные сохраняются и используются после повторного включения.

### Раунд 2

- **Storage toggle**: флаг ON/OFF хранится в storage соответствующей фичи, не в `local-settings`.
- **Visual QA**: отдельные состояния ON/OFF для обеих фич должны быть доступны в Storybook.
