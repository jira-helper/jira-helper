# Request: Улучшение диагностики через DI для сбора данных от фич

**Дата**: 2026-05-14
**Slug**: diagnostic-data-collection
**Тип**: logic
**Has UI**: no
**Scope**: medium

## Описание запроса

Нужна новая фича - хочу сделать диагностику более удобной. Конкретно модуль диагностики должен провайдить в DI систему токен типа "добавь данные для диагностики" - это какая то функцию, которую, если дернуть, можно получить данные для диагностики в этот момент.

Все фичи подключают этот DI и регистрируют данные.

При выгрузке диагностических данных вызывается этот di и собирается суммирующий отчет. Это в добавление к тому что уже реализовано в диагностике.

Если вызов сбора диагностических данных упал - сама диагностика упасть не должна - ошибку сбора тоже кладем в диагностику как лог или как отдельное поле.

## Уточнения

### Раунд 1

**Формат данных**: Фичи возвращают plain object (структура с полями) для сериализации в JSON.

**Механизм регистрации**: Фичи регистрируют callback для сбора данных + ключ.

**Scope фич**: В первой версии затронуть все фичи которые используют boardProperty или настройки в localStorage.

### Раунд 2

**API регистрации**: Фича вызывает registerDiagnosticData('featureName', () => Result<{...}>)

**Время регистрации**: При инициализации модуля (в методе register() модуля фичи)

**Обработка ошибок**: Ловим throw от callback, ожидаем что callback возвращает Result<T> (Ok/Err)
