# User Doc Writer

## Назначение

Создать файлы `user-guide.md` (EN) и `user-guide.ru.md` (RU) в директории `docs/` внутри модуля фичи (`src/features/<module>/docs/`). Документация следует фиксированному формату.

## Где создавать

```
src/features/<module>/
├── docs/
│   ├── user-guide.md      # EN
│   └── user-guide.ru.md   # RU
```

## Формат

Каждый файл начинается с таблицы характеристик, затем идут фиксированные секции в строгом порядке:

```md
# Название фичи

| | |
|---|---|
| Где настраивается | Board Settings → вкладка → точный путь |
| Где видно | Board (detail view) / Issue Page / Reports |
| Настройки действуют | Jira Board Property (`propertyKey`) или localStorage (`storageKey`) |

## Цель

Зачем нужна эта фича, какую проблему решает. Одно-два предложения.

## Как настроить

Пошаговая инструкция:
1. конкретный шаг
2. конкретный шаг
...

## Как использовать

Что пользователь видит на доске / странице задачи после настройки. Как интерпретировать визуальные элементы. Цветовые коды, бейджи, их значения.

## Сценарии использования

Конкретные примеры с контекстом:
- «Я хочу ограничить WIP на группу колонок "In Progress + Review" до 5 задач»
- «Я хочу видеть прогресс подзадач на карточках, сгруппированный по исполнителям»

## Устранение неполадок

Распространённые проблемы и их решения.

## См. также

- [Связанная фича](ссылка)
- [Общие настройки](ссылка)
```

## Правила

### Таблица характеристик

- 3 строки, всегда в начале
- **Где настраивается**: точный путь в UI (Board Settings → вкладка → кнопка/секция)
- **Где видно**: Board / Issue Page / Reports + уточнение (detail view / backlog)
- **Хранилище**: Jira Board Property с ключом (в кавычках) ИЛИ localStorage с ключом

### Язык

- EN версия: простой технический английский, без жаргона
- RU версия: полный перевод EN-версии. Не сокращать. Профессиональный технический русский.
- Сохранять названия UI-элементов как в Jira (Board Settings, Columns, Save)

### Секции

- Все 6 секций обязательны
- Порядок фиксированный
- Не добавлять новые секции
- Если фича zero-config — «Как настроить» = «Настройка не требуется, фича работает автоматически»

### Сценарии

- Минимум 1, максимум 3 сценария
- Каждый начинается с «Я хочу...» (в RU: «Я хочу...», в EN: «I want to...»)
- Описывать конкретную ситуацию и ожидаемый результат

### Ссылки

- В «См. также» — ссылки на связанные фичи в формате относительного пути от docs/features/
- Всегда добавлять ссылку на [Общие настройки](/docs/settings)

## Процесс

1. Прочитай код фичи: `module.ts`, `types.ts`, `tokens.ts`, settings components — чтобы понять точные UI-пути и ключи хранилища
2. Если в модуле есть `feature.md` / `feature.ru.md` — используй их как источник
3. Напиши `docs/user-guide.md` (EN)
4. Напиши `docs/user-guide.ru.md` (RU) — полный перевод
