## Предусловия

## 4. Issue Condition Check (Иконки по условиям)

### 4.1 Настройки (Settings)

- [ ] **TC-IC-S01: Включение фичи**

  4. Ввести JQL (например, `status = "In Progress"`)
  5. Выбрать иконку (например, warning)

  7. Сохранить → условие добавлено в список

  1. Добавить новое условие
  2. Выбрать режим "With subtasks" → появляются поля для issue JQL и subtask JQL
  3. Ввести JQL для задачи (issueJql) и JQL для подзадач (subtaskJql)
  4. Выбрать иконку и тултип
  5. Сохранить → условие добавлено

- [ ] **TC-IC-S04: Выбор иконки**
  1. Открыть селектор иконок → доступно 30+ иконок

  3. Каждая иконка отображается в предпросмотре

- [ ] **TC-IC-S05: Выбор цвета фона**
  1. Открыть color picker
  2. Выбрать цвет из рекомендованных (#ffebee, #e3f2fd) → цвет применён
  3. Выбрать яркий цвет (#ff4d4f) → цвет фона бейджа изменился
  4. Убрать цвет (без фона) → бейдж без цветного фона

- [ ] **TC-IC-S06: Настройка анимации**
  1. Выбрать анимацию "None" → бейдж статичный
  2. Выбрать "Pulse" → бейдж пульсирует
  3. Проверить "Breathe", "Blink", "Blink Fast", "Shake" → каждая анимация визуально отличается

- [ ] **TC-IC-S07: Включение/выключение условия**
  1. Создать условие (enabled = true) → условие активно

  3. Включить обратно → бейдж снова появляется

- [ ] **TC-IC-S09: Валидация обязательных полей**

  2. Оставить tooltip пустым → ошибка "Tooltip text is required"

- [ ] **TC-IC-S10: Настройка источников подзадач (withSubtasks)**

  4. Выключить все источники → условие не матчит

- [ ] **TC-IC-S11: Режим матчинга подзадач**

  2. Выбрать subtaskMatchMode = "all" → все подзадачи должны соответствовать

  2. Обновить доску → бейдж больше не отображается
