# Сбор требований

Этот skill описывает процесс сбора требований для новой фичи.

## Когда использовать

- Создание новой фичи
- Существенная переработка существующей фичи
- Неясные или неполные требования от пользователя

## Обязательные вопросы

### 1. Пользователь и контекст

```
Кто будет использовать эту фичу?
- [ ] Team Lead
- [ ] Developer
- [ ] Scrum Master
- [ ] Все участники команды
```

```
Какую проблему решаем?
[Опиши проблему в 1-2 предложениях]
```

### 2. Данные и хранение

```
Где хранятся данные фичи?
- [ ] Jira Board Property (синхронизируется между пользователями)
- [ ] Локально в браузере (только для текущего пользователя)
- [ ] Runtime (не сохраняется, только в памяти)
- [ ] Комбинация (уточни какие данные где)
```

```
Какая структура данных?
[Опиши основные поля или дай пример JSON]
```

### 3. UI и взаимодействие

```
Какие экраны затрагиваются?
- [ ] Settings Page (настройки доски)
- [ ] Board Page (сама доска с карточками)
- [ ] Backlog Page
- [ ] Другое: ___
```

```
Что видит пользователь?
[Опиши основные UI-элементы]
```

```
Какие действия доступны?
[Опиши основные операции: добавить, редактировать, удалить, etc.]
```

### 4. Бизнес-правила

```
Какие ограничения и валидации?
[Например: минимальное значение, обязательные поля]
```

```
Как обрабатываются edge cases?
- Пустые данные: ___
- Конфликты: ___
- Ошибки сети: ___
```

### 5. Интеграции

```
С какими существующими фичами интегрируется?
[Например: column-limits, person-limits, swimlanes]
```

```

- [ ] Board Properties (getBoardProperty/setBoardProperty)
- [ ] Rapid View Config
- [ ] Issue Search (JQL)
- [ ] User API
- [ ] Другое: ___
```

### 6. Приоритеты

```

## Пример диалога

**Агент**: Расскажи подробнее о фиче. Кто будет её использовать?

**Пользователь**: Team Lead. Хочу видеть сколько задач в каждой колонке и ограничивать максимум.

**Агент**: Понял. Где должны храниться настройки лимитов?

**Пользователь**: В настройках доски, чтобы все видели.

**Агент**: Значит Jira Board Property. Какие поля нужны для лимита?

**Пользователь**: ID колонки и число — максимум задач.

**Агент**: 
```typescript
type ColumnLimit = {
  columnId: string;
  limit: number;
};
```
Верно? Какие еще поля могут понадобиться?
